3/*
4 From NanoVDB.h
5
6 Memory layout:
7
8 Notation: "]---[" implies it has optional padding, and "][" implies zero padding
9
10 [GridData(672B)][TreeData(64B)]---[RootData][N x Root::Tile]---[NodeData<5>]---[ModeData<4>]---[LeafData<3>]---[BLINDMETA...]---[BLIND0]---[BLIND1]---etc.
11 ^ ^ ^ ^ ^ ^
12 | | | | | |
13 +-- Start of 32B aligned buffer | | | | +-- Node0::DataType* leafData
14 GridType::DataType* gridData | | | |
15 | | | +-- Node1::DataType* lowerData
16 RootType::DataType* rootData --+ | |
17 | +-- Node2::DataType* upperData
18 |
19 +-- RootType::DataType::Tile* tile
20
21
22 It's important to emphasize that all the grid data (defined below) are explicitly 32 byte
23 aligned, which implies that any memory buffer that contains a NanoVDB grid must also be at
24 32 byte aligned. That is, the memory address of the beginning of a buffer (see ascii diagram below)
25 must be divisible by 32, i.e. uintptr_t(&buffer)%32 == 0! If this is not the case, the C++ standard
26 says the behaviour is undefined! Normally this is not a concerns on GPUs, because they use 256 byte
27 aligned allocations, but the same cannot be said about the CPU.
28
29 GridData is always at the very beginning of the buffer immediately followed by TreeData!
30 The remaining nodes and blind-data are allowed to be scattered throughout the buffer,
31 though in practice they are arranged as:
32
33 GridData: 672 bytes (e.g. magic, checksum, major, flags, index, count, size, name, map, world bbox, voxel size, class, type, offset, count)
34
35 TreeData: 64 bytes (node counts and byte offsets)
36
37 ... optional padding ...
38
39 RootData: size depends on ValueType (index bbox, voxel count, tile count, min/max/avg/standard deviation)
40
41 Array of: RootData::Tile
42
43 ... optional padding ...
44
45 Array of: Upper InternalNodes of size 32^3: bbox, two bit masks, 32768 tile values, and min/max/avg/standard deviation values
46
47 ... optional padding ...
48
49 Array of: Lower InternalNodes of size 16^3: bbox, two bit masks, 4096 tile values, and min/max/avg/standard deviation values
50
51 ... optional padding ...
52
53 Array of: LeafNodes of size 8^3: bbox, bit masks, 512 voxel values, and min/max/avg/standard deviation values
54*/
