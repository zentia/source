32/*
33 * The PoolEvolution pattern models the evolution of a population.
34 * In the pattern, a “candidate selection” function (s) selects a subset of objects belonging
35 * to an unstructured object pool (P).
36 * The selected objects are processed by means of an “evolution” function (e).
37 * The evolution function may produce any number of new/modified objects out of the input one.
38 * The set of objects computed by the evolution function on the selected object are filtered
39 * through a “filter” function (f) and eventually inserted into the object pool.
40 * At any insertion/extraction into/from the object pool a “termination” function (t) is
41 * evaluated on the object pool, to determine whether the evolution process has to be stopped or
42 * continued for further iterations.
43 * A pool evolution pattern therefore computes P as result of the following algorithm:
44 *
45 * while not( t(P) ) do
46 * N = e ( s(P) )
47 * P += f (N, P)
48 * end while
49 *
50 */
