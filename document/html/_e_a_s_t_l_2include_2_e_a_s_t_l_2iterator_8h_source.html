347 // The C++ library working group has tentatively approved the usage of two
348 // template parameters (Iterator1 and Iterator2) in order to allow reverse_iterators
349 // and const_reverse iterators to be comparable. This is a similar issue to the
350 // C++ defect report #179 regarding comparison of container iterators and const_iterators.
351 //
352 // libstdc++ reports that std::relops breaks the usage of two iterator types and if we
353 // want to support relops then we need to also make versions of each of below with
354 // a single template parameter to placate std::relops. But relops is hardly used due to
355 // the troubles it causes and so we are avoiding support here until somebody complains about it.

358 operator==(const reverse_iterator<Iterator1>& a, const reverse_iterator<Iterator2>& b)
359 { return a.base() == b.base(); }

875 {
