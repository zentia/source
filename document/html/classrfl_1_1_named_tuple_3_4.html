◆ values_

Values rfl::NamedTuple< FieldTypes >::values_
private

The values actually contained in the named tuple. As you can see, a NamedTuple is just a normal tuple under-the-hood, everything else is resolved at compile time. It should have no runtime overhead over a normal rfl::Tuple.
