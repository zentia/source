7// Specification
8//
9// This file implements C++ type traits as proposed by the emerging C++ update
10// as of May, 2005. This update is known as "Proposed Draft Technical Report
11// on C++ Library Extensions" and is document number n1745. It can be found
12// on the Internet as n1745.pdf and as of this writing it is updated every
13// couple months to reflect current thinking.
15
16
18// Description
19//
20// EASTL includes a fairly serious type traits library that is on par with the
21// one found in Boost but offers some additional performance-enhancing help as well.
22// The type_traits library provides information about class types, as opposed to
23// class instances. For example, the is_integral type trait tells if a type is
24// one of int, short, long, char, uint64_t, etc.
25//
26// There are three primary uses of type traits:
27// * Allowing for optimized operations on some data types.
28// * Allowing for different logic pathways based on data types.
29// * Allowing for compile-type assertions about data type expectations.
30//
31// Here is an example of using type traits to tell if a value is a floating point
32// value or not:
33//
34// template <typename T>
35// DoSomething(T t) {
36// assert(is_floating_point<T>::value);
37// }
38//
39// In this example, if Widget is trivally copyable.
40// vector<Widget> wVector;
41// .. add elements ...
42// wVector.erase(wVector.begin());
43// The vector::erase() operation will optimize the moving of [begin() + 1, end()) to [begin(), end())
44// by using memcpy instead of explicitly calling the copy constructor of each element.
45//
46// The following is a full list of the currently recognized type traits. Most of these
47// are implemented as of this writing, but if there is one that is missing, feel free
48// to contact the maintainer of this library and request that it be completed.
49// As of this writing all C++11 type traits are supported, as well as some additional ones.
50// http://en.cppreference.com/w/cpp/types
51//
52// Trait Description
53// ------------------------------------------------------------------------------
54// is_void T is void or a cv-qualified (const/void-qualified) void.
55// is_null_pointer
56// is_integral T is an integral type.
57// is_floating_point T is a floating point type.
58// is_array T is an array type. The templated array container is not an array type.
59// is_enum T is an enumeration type.
60// is_union T is a union type.
61// is_class T is a class type but not a union type.
62// is_function T is a function type.
63// is_pointer T is a pointer type. Includes function pointers, but not pointers to (data or function) members.
64// is_rvalue_reference
65// is_lvalue_reference
66// is_member_object_pointer T is a pointer to data member.
67// is_member_function_pointer T is a pointer to member function.
68//
69// is_fundamental T is a fundamental type (void, integral, or floating point).
70// is_arithmetic T is an arithmetic type (integral or floating point).
71// is_scalar T is a scalar type (arithmetic, enum, pointer, member_pointer)
72// is_object T is an object type.
73// is_compound T is a compound type (anything but fundamental).
74// is_reference T is a reference type. Includes references to functions.
75// is_member_pointer T is a pointer to a member or member function.
76//
77// is_const T is const-qualified.
78// is_volatile T is volatile-qualified.
79// is_trivial
80// is_trivially_copyable
81// is_standard_layout
82// is_pod T is a POD type.
83// is_empty T is an empty class.
84// is_polymorphic T is a polymorphic class.
85// is_abstract T is an abstract class.
86// is_signed T is a signed integral type.
87// is_unsigned T is an unsigned integral type.
88// is_bounded_array T is a type is an array type of known bound
89// is_unbounded_array T is a type is an array type of unknown bound

91// is_constructible
92// is_trivially_constructible
93// is_nothrow_constructible
94// is_default_constructible
95// is_trivially_default_constructible
96// is_nothrow_default_constructible

103// is_assignable

114// is_nothrow_destructible
115// has_virtual_destructor T has a virtual destructor.

117// alignment_of An integer value representing the number of bytes of the alignment of objects of type T; an object of type T may be allocated at an address that is a multiple of its alignment.
118// rank An integer value representing the rank of objects of type T. The term 'rank' here is used to describe the number of dimensions of an array type.
119// extent An integer value representing the extent (dimension) of the I'th bound of objects of type T. If the type T is not an array type, has rank of less than I, or if I == 0 and T is of type 'array of unknown bound of U,' then value shall evaluate to zero; otherwise value shall evaluate to the number of elements in the I'th array bound of T. The term 'extent' here is used to describe the number of elements in an array type.

No function-parameter adjustments are made to type To when determining whether From is convertible to To; this implies that if type To is a function type or an array type, then the condition is false.

171// detected_or_t Equivalent to detected_or<Default, Op, Args...>::type.
172// is_detected_exact Check that the type we obtain after expanding some arguments (Args) over a constraint (Op) is equivalent to Expected.
173// is_detected_convertible Check that the type we obtain after expanding some arguments (Args) over a constraint (Op) is convertible to Expected.
174//
175// Deprecated pre-C++11 type traits
176// add_reference Deprecated in favor of add_lvalue_reference(_t).
177// add_signed Deprecated in favor of make_signed.
178// add_unsigned Deprecated in favor of make_unsigned.
179// identity Simply sets T as type. Use type_identity(_t) instead.
180// is_array_of_known_bounds Deprecated in favor of is_bounded_array.
181// is_array_of_unknown_bounds Deprecated in favor of is_unbounded_array.
182// has_trivial_constructor The default constructor for T is trivial.
183// has_trivial_copy The copy constructor for T is trivial.
184// has_trivial_assign The assignment operator for T is trivial.
185// has_trivial_destructor The destructor for T is trivial.
186// has_nothrow_constructor The default constructor for T has an empty exception specification or can otherwise be deduced never to throw an exception.
187// has_nothrow_copy The copy constructor for T has an empty exception specification or can otherwise be deduced never to throw an exception.
188// has_nothrow_assign The assignment operator for T has an empty exception specification or can otherwise be deduced never to throw an exception.
189// *has_trivial_relocate T can be moved to a new location via bitwise copy. Note that C++11 rvalue/move functionality supercedes this.
190//
191// Deprecated in C++17
192// is_literal_type Deprecated: use constexpr if to determine whether an expression is constant evaluated.
193// result_of Deprecated: use invoke_result instead.

201// Requirements
202//
203// As of this writing (5/2005), type_traits here requires a well-conforming
204// C++ compiler with respect to template metaprogramming. To use this library
205// you need to have at least one of the following:
206// MSVC++ 7.1 (includes Win32, Win64, and WinCE platforms)
207// GCC 3.2 (includes MacOSX, and Linux platforms)
208// Metrowerks 8.0 (incluees MacOSX, Windows, and other platforms)
209// EDG (includes any compiler with EDG as a back-end, such as the Intel compiler)
210// Comeau (this is a C++ to C generator)
211//
212// It may be useful to list the compilers/platforms the current version of
213// type_traits doesn't support:
214// Borland C++ (it simply has too many bugs with respect to templates).
215// GCC 2.96 We used to have a separate set of type traits for this compiler, but removed it due to lack of necessity.
217
219// Implementation
220//
221// The implementation here is almost entirely based on template metaprogramming.
222// This is whereby you use the compiler's template functionality to define types
223// and values and make compilation decisions based on template declarations.
224// Many of the algorithms here are similar to those found in books such as
225// "Modern C++ Design" and C++ libraries such as Boost. The implementations here
226// are simpler and more straightforward than those found in some libraries, due
227// largely to our assumption that the compiler is good at doing template programming.

250 // integral_constant
251 //
252 // This is the base class for various type traits, as defined by C++11.
253 // This is essentially a utility base class for defining properties
254 // as both class constants (value) and as types (type).
255 //
256 template <typename T, T v>
257 struct integral_constant
258 {
259 static EA_CONSTEXPR T value = v;
260 typedef T value_type;
261 typedef integral_constant<T, v> type;
262
263 EA_CONSTEXPR operator value_type() const EA_NOEXCEPT { return value; }
264 EA_CONSTEXPR value_type operator()() const EA_NOEXCEPT { return value; }
265 };

639 // remove_const
640 //
641 // Remove const from a type.
642 //
643 // The remove_const transformation trait removes top-level const
644 // qualification (if any) from the type to which it is applied. For a
645 // given type T, remove_const<T const>::type is equivalent to the type T.
646 // For example, remove_const<char*>::type is equivalent to char* while
647 // remove_const<const char*>::type is equivalent to const char*.
648 // In the latter case, the const qualifier modifies char, not *, and is
649 // therefore not at the top level.
650 //
652
653 #define EASTL_TYPE_TRAIT_remove_const_CONFORMANCE 1 // remove_const is conforming.
654
655 template <typename T> struct remove_const { typedef T type; };
656 template <typename T> struct remove_const<const T> { typedef T type; };
657 template <typename T> struct remove_const<const T[]> { typedef T type[]; };
658 template <typename T, size_t N> struct remove_const<const T[N]> { typedef T type[N]; };
659
660 #if EASTL_VARIABLE_TEMPLATES_ENABLED
661 template<typename T>
662 using remove_const_t = typename remove_const<T>::type;
663 #endif

666 // remove_volatile
667 //
668 // Remove volatile from a type.
669 //
670 // The remove_volatile transformation trait removes top-level volatile
671 // qualification (if any) from the type to which it is applied.
672 // For a given type T, the type remove_volatile <T volatile>::T is equivalent
673 // to the type T. For example, remove_volatile <char* volatile>::type is
674 // equivalent to char* while remove_volatile <volatile char*>::type is
675 // equivalent to volatile char*. In the latter case, the volatile qualifier
676 // modifies char, not *, and is therefore not at the top level.
677 //
679
680 #define EASTL_TYPE_TRAIT_remove_volatile_CONFORMANCE 1 // remove_volatile is conforming.
681
682 template <typename T> struct remove_volatile { typedef T type; };

695 //
696 // Remove const and volatile from a type.
697 //
698 // The remove_cv transformation trait removes top-level const and/or volatile
699 // qualification (if any) from the type to which it is applied. For a given type T,
