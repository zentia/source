inlinestatic

Gets a clone of the attachment remapped with a sprite image.

返回
The remapped clone.
参数
oThe original attachment.
spriteThe sprite whose texture to use.
sourceMaterialThe source material used to copy the shader and material properties from.
premultiplyAlphaIf true, a premultiply alpha clone of the original texture will be created. See remarks below for additional info.
cloneMeshAsLinkedIf true MeshAttachments will be cloned as linked meshes and will inherit animation from the original attachment.
useOriginalRegionSizeIf true the size of the original attachment will be followed, instead of using the Sprite size.
pivotShiftsMeshUVCoordsIf true and the original Attachment is a MeshAttachment, then a non-central sprite pivot will shift uv coords in the opposite direction. Vertices will not be offset in any case when the original Attachment is a MeshAttachment.
useOriginalRegionScaleIf true and the original Attachment is a RegionAttachment, then the original region's scale value is used instead of the Sprite's pixels per unit property. Since uniform scale is used, x scale of the original attachment (width scale) is used, scale in y direction (height scale) is ignored.
pmaCloneTextureFormatIf premultiplyAlpha is true>, the TextureFormat of the newly created PMA attachment Texture.
pmaCloneMipmapsIf premultiplyAlpha is <ctrue>, whether the newly created PMA attachment Texture has mipmaps enabled.

When parameter premultiplyAlpha is set to true, a premultiply alpha clone of the original texture will be created. Additionally, this PMA Texture clone is cached for later re-use, which might steadily increase the Texture memory footprint when used excessively. See AtlasUtilities.ClearCache() on how to clear these cached textures.
