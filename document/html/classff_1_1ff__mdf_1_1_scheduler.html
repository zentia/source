EOS callback

This method is called when an EOS has just been received from one input channel. Inside this method it is possible to call ff_send_out to produce data elements in output (this is not possible in the svc_end method). The parameter

参数
idis the ID of the channel that received the EOS.

Service initialisation

Called after run-time initialisation (e.g. thread spawning) but before to start to get items from input stream (can be useful for initialisation of parallel activities, e.g. manual thread pinning that cannot be done in the costructor because threads stil do not exist).
