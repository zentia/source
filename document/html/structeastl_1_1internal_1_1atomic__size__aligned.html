◆ EA_ALIGN()

template<typename T>
eastl::internal::atomic_size_aligned< T >::EA_ALIGN ( sizeof(T) )
private

Some compilers such as MSVC will align 64-bit values on 32-bit machines on 4-byte boundaries which can ruin the atomicity guarantees.

Ensure everything is size aligned.

mutable is needed in cases such as when loads are only guaranteed to be atomic using a compare exchange, such as for 128-bit atomics, so we need to be able to have write access to the variable as one example.
