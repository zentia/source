In my own opinion, I found the wording on Microsoft docs a little confusing. ExchangeHigh means the top 8 bytes so (ptr + 8). ExchangeLow means the low 8 butes so (ptr). Endianness does not matter since we are just loading data and comparing data. Thought of as memcpy() and memcmp() function calls whereby the layout of the data itself is irrelevant. Only after we type pun back to the original type, and load from memory does the layout of the data matter again.

Unfortunately MSVC Intrinsics depend on the architecture that we are compiling for. These are some indirection macros to make our lives easier and ensure the least possible amount of copy-paste to reduce programmer errors.

All compiler implementations end up deferring to the below macros.
