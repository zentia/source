class cds\+::urcu\+::general\+\_\+threaded$<$ Buffer, Lock, Disposer\+Thread, Backoff $>$}
User-\/space general-\/purpose RCU with deferred threaded reclamation 

This implementation is similar to \doxylink{classcds_1_1urcu_1_1general__buffered}{general\+\_\+buffered} but separate thread is created for deleting the retired objects. Like {\ttfamily general\+\_\+buffered}, the class contains an internal buffer where retired objects are accumulated. When the buffer becomes full, the RCU {\ttfamily \doxylink{classcds_1_1urcu_1_1general__threaded_addd92691234888a31059b578821d272a}{synchronize()}} function is called that waits until all reader/updater threads end up their read-\/side critical sections, i.\+e. until the RCU quiescent state will come. After that the "{}work ready"{} message is sent to reclamation thread. The reclamation thread frees the buffer. This synchronization cycle may be called in any thread that calls {\ttfamily \doxylink{classcds_1_1urcu_1_1general__threaded_a3235164a97b7818e3b57c69e174c8e28}{retire\+\_\+ptr()}} function.

There is a wrapper \doxylink{classcds_1_1urcu_1_1gc_3_01general__threaded_3_01_buffer_00_01_lock_00_01_disposer_thread_00_01_backoff_01_4_01_4_cds_urcu_general_threaded_gc}{gc\texorpdfstring{$<$}{<}general\+\_\+threaded\texorpdfstring{$>$}{>}} for {\ttfamily general\+\_\+threaded} class that provides unified RCU interface. You should use this wrapper class instead {\ttfamily general\+\_\+threaded} 

The {\ttfamily \doxylink{struct_buffer}{Buffer}} contains items of \doxylink{namespacecds_1_1urcu_cds_urcu_retired_ptr}{epoch\+\_\+retired\+\_\+ptr} type

and it should support a multiple producer/single consumer queue with the following interface\+:
\begin{DoxyItemize}
\item {\ttfamily  bool push( epoch\+\_\+retired\+\_\+ptr\& p ) } -\/ places the retired pointer {\ttfamily p} into queue. If the function returns {\ttfamily false} it means that the buffer is full and RCU synchronization cycle must be processed.
\item {\ttfamily \doxylink{structcds_1_1urcu_1_1epoch__retired__ptr}{epoch\+\_\+retired\+\_\+ptr} \texorpdfstring{$\ast$}{*} front() } -\/ returns a pointer to the top element or {\ttfamily nullptr} if the buffer is empty.
\item {\ttfamily bool pop\+\_\+front() } -\/ pops the top element; returns {\ttfamily false} if the buffer is empty.
\item {\ttfamily size\+\_\+t \doxylink{imgui__impl__opengl3__loader_8h_ac42c05b5701aef6bab82675cf3b55fee}{size()}} -\/ returns queue\textquotesingle{}s item count.
\end{DoxyItemize}

The buffer is considered as full if {\ttfamily \doxylink{test__concurrent__queue_8cpp_a262227aa72b9cc1db7b9a80d7f295211}{push()}} returns {\ttfamily false} or the buffer size reaches the RCU threshold.

Template arguments\+:
\begin{DoxyItemize}
\item {\ttfamily \doxylink{struct_buffer}{Buffer}} -\/ MPSC (muliple producer/single consumer) buffer type with FIFO semantics.

Default is {\ttfamily \doxylink{namespacecds_1_1container_af611ea3db0802913eb8dafdb6dda8c0f}{cds\+::container\+::\+Vyukov\+MPSCCycle\+Queue}}. The buffer contains the objects of \doxylink{structcds_1_1urcu_1_1epoch__retired__ptr}{epoch\+\_\+retired\+\_\+ptr} type that contains additional {\ttfamily m\+\_\+n\+Epoch} field. This field specifies an epoch when the object has been placed into the buffer. The {\ttfamily general\+\_\+threaded} object has a global epoch counter that is incremented on each {\ttfamily \doxylink{classcds_1_1urcu_1_1general__threaded_addd92691234888a31059b578821d272a}{synchronize()}} call. The epoch is used internally to prevent early deletion.
\item {\ttfamily Lock} -\/ mutex type, default is {\ttfamily std\+::mutex} 
\item {\ttfamily Disposer\+Thread} -\/ the reclamation thread class. Default is \doxylink{classcds_1_1urcu_1_1dispose__thread}{cds\+::urcu\+::dispose\+\_\+thread}, see the description of this class for required interface.
\item {\ttfamily Backoff} -\/ back-\/off schema, default is \doxylink{namespacecds_1_1backoff_a0f64f7bc371e3ad006fba5bd855d6e96}{cds\+::backoff\+::\+Default} 
\end{DoxyItemize}
