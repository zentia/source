\subsubsection*{template$<$typename GC, typename \mbox{\hyperlink{_tutorial__sparse__example_8cpp_af639fb96631ba0018f0b2259fc95edfb}{T}}, typename Traits$>$\newline
class cds\+::container\+::\+Iterable\+List$<$ GC, T, Traits $>$}
Iterable ordered list 

\label{classcds_1_1container_1_1_iterable_list_cds_nonintrusive_IterableList_gc}%
\Hypertarget{classcds_1_1container_1_1_iterable_list_cds_nonintrusive_IterableList_gc}%
 This lock-\/free list implementation supports thread-\/safe iterators.

Usually, ordered single-\/linked list is used as a building block for the hash table implementation. Iterable list is suitable for almost append-\/only hash table because the list doesn\textquotesingle{}t delete its internal node when erasing a key but it is marked them as empty to be reused in the future. However, plenty of empty nodes degrades performance.

The complexity of searching is {\ttfamily O(\+N)}.

Template arguments\+:
\begin{DoxyItemize}
\item {\ttfamily GC} -\/ Garbage collector used.
\item {\ttfamily T} -\/ type to be stored in the list.
\item {\ttfamily Traits} -\/ type traits, default is {\ttfamily \doxylink{structcds_1_1container_1_1iterable__list_1_1traits}{iterable\+\_\+list\+::traits}}.
\end{DoxyItemize}

Unlike standard container, this implementation does not divide type {\ttfamily T} into key and value part and may be used as a main building block for hash set algorithms. The key is a function (or a part) of type {\ttfamily T}, and this function is specified by {\ttfamily Traits\+::compare} functor or {\ttfamily Traits\+::less} predicate.

{\ttfamily \doxylink{classcds_1_1container_1_1_iterable_k_v_list}{Iterable\+KVList}} is a key-\/value version of iterable non-\/intrusive list that is closer to the C++ std library approach.

Forward iterator 

The forward iterator for iterable list has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item to protect the value, the iterator contains a GC-\/specific guard. For some GC (like as {\ttfamily \doxylink{namespacecds_1_1gc_af0523881f089593a7559d194ee2b5dd9}{gc\+::\+HP}}), a guard is a limited resource per thread, so an exception (or assertion) "{}no free guard"{} may be thrown if the limit of guard count per thread is exceeded.
\item The iterator cannot be moved across thread boundary since it contains thread-\/private GC\textquotesingle{}s guard.
\item Iterator is thread-\/safe\+: even if an element the iterator points to is removed, the iterator stays valid because it contains the guard keeping the value from to be recycled.
\end{DoxyItemize}
