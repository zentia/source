\mbox{[}type-\/option\mbox{]} Option setter for node allocator 

{\ttfamily \doxylink{ir__common_8h_a1ac75f8fee198af783521627f1fecfbc}{Type}} is allocator with {\ttfamily std\+::allocator} interface. Default is value of macro CDS\+\_\+\+DEFAULT\+\_\+\+ALLOCATOR that, in turn, is {\ttfamily std\+::allocator}.

Many node-\/base containers require an allocator for maintaining data (container\textquotesingle{}s node) and for internal use. Sometimes, this types of allocator should be different for performance reason. For example, we should like to allocate the node from a pool of preallocated nodes. Such pool can be seen as the node allocator.

Usually, if a container supports {\ttfamily \doxylink{structcds_1_1opt_1_1allocator}{opt\+::allocator}} and {\ttfamily opt\+::node\+\_\+allocator} options and {\ttfamily \doxylink{structcds_1_1opt_1_1node__allocator}{opt\+::node\+\_\+allocator}} is not specified the {\ttfamily opt\+::allocator} option is used for maintaining the nodes.

The {\ttfamily libcds} containers actively use rebinding to convert an allocator of one type to another. Thus, you may specify any valid type as std\+::allocator\textquotesingle{}s template parameter.
