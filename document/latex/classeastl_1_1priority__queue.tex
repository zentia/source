class eastl\+::priority\+\_\+queue$<$ T, Container, Compare $>$}
\doxylink{classeastl_1_1priority__queue}{priority\+\_\+queue}

The behaviour of this class is just like the std\+::priority\+\_\+queue class and you can refer to std documentation on it.

A \doxylink{classeastl_1_1priority__queue}{priority\+\_\+queue} is an adapter container which implements a queue-\/like container whereby \doxylink{classeastl_1_1priority__queue_a53bc5de9fc58ff8e0974d8c592ac064f}{pop()} returns the item of highest priority. The entire queue isn\textquotesingle{}t necessarily sorted; merely the first item in the queue happens to be of higher priority than other items. You can read about priority\+\_\+queues in many books on algorithms, such as "{}\+Algorithms"{} by Robert Sedgewick.

The Container type is a container which is random access and supports \doxylink{classeastl_1_1priority__queue_a840cdbde6721b9c52a096d0694c3e16d}{empty()}, \doxylink{imgui__impl__opengl3__loader_8h_a3d1e3edfcf61ca2d831883e1afbad89e}{size()}, clear(), insert(), front(), push\+\_\+back(), and pop\+\_\+back(). You would typically use vector or deque.

Note that we don\textquotesingle{}t provide functions in the \doxylink{classeastl_1_1priority__queue}{priority\+\_\+queue} interface for working with allocators or names. The reason for this is that \doxylink{classeastl_1_1priority__queue}{priority\+\_\+queue} is an adapter class which can work with any standard sequence and not necessarily just a sequence provided by this library. So what we do is provide a member accessor function \doxylink{classeastl_1_1priority__queue_ac85845b05da6c9f54f4297545018f96e}{get\+\_\+container()} which allows the user to manipulate the sequence as needed. The user needs to be careful not to change the container\textquotesingle{}s contents, however.

Classic heaps allow for the concept of removing arbitrary items and changing the priority of arbitrary items, though the C++ std heap (and thus \doxylink{classeastl_1_1priority__queue}{priority\+\_\+queue}) functions don\textquotesingle{}t support these operations. We have extended the heap algorithms and the \doxylink{classeastl_1_1priority__queue}{priority\+\_\+queue} implementation to support these operations.
