Before you can contribute, \doxylink{namespace_e_a}{EA} must have a Contributor License Agreement (CLA) on file that has been signed by each contributor. You can sign here\+: \href{https://electronicarts.na1.echosign.com/public/esignWidget?wid=CBFCIBAA3AAABLblqZhByHRvZqmltGtliuExmuV-WNzlaJGPhbSRg2ufuPsM3P0QmILZjLpkGslg24-UJtek*}{\texttt{ Go to CLA}}

If you want to be recognized for your contributions to EASTL or have a project using EASTL be recognized; you can submit a pull request to the appropriate sections in README.md. Some examples of what the format and information will look like is as follows.

\end{DoxyItemize}\hypertarget{md_runtime_2_e_a_s_t_l_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md1}{}\doxysubsection{\texorpdfstring{Pull Request Policy}{Pull Request Policy}}\label{md_runtime_2_e_a_s_t_l_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md1}
All code contributions to EASTL are submitted as \href{https://help.github.com/articles/using-pull-requests/}{\texttt{ Github pull requests}}. All pull requests will be reviewed by an EASTL maintainer according to the guidelines found in the next section.

Your pull request should\+:


\begin{DoxyItemize}
\item merge cleanly
\item come with tests
\begin{DoxyItemize}
\item tests should be minimal and stable
\item fail before your fix is applied
\end{DoxyItemize}
\item pass the test suite
\item code formatting is encoded in clang format
\begin{DoxyItemize}
\item limit using clang format on new code
\item do not deviate from style already established in the files
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md_runtime_2_e_a_s_t_l_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md2}{}\doxysubsection{\texorpdfstring{Getting the Repository}{Getting the Repository}}\label{md_runtime_2_e_a_s_t_l_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md2}
EASTL uses git submodules for its dependencies as they are seperate git repositories. Recursive clones will continue until HD space is exhausted unless they are manually limited. It is recommended to use the following to get the source\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https://github.com/electronicarts/EASTL}
\DoxyCodeLine{cd\ EASTL}
\DoxyCodeLine{git\ submodule\ update\ -\/-\/init}

\end{DoxyCode}
\hypertarget{md_runtime_2_e_a_s_t_l_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md3}{}\doxysubsection{\texorpdfstring{Running the Unit Tests}{Running the Unit Tests}}\label{md_runtime_2_e_a_s_t_l_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md3}
EASTL uses CMake as its build system.


\begin{DoxyItemize}
\item Create and navigate to "{}your\+\_\+build\+\_\+folder"{}\+:
\begin{DoxyItemize}
\item mkdir your\+\_\+build\+\_\+folder \&\& cd your\+\_\+build\+\_\+folder
\end{DoxyItemize}
\item Generate build scripts\+:
\begin{DoxyItemize}
\item cmake eastl\+\_\+source\+\_\+folder -\/DEASTL\+\_\+\+BUILD\+\_\+\+TESTS\+:BOOL=ON
\end{DoxyItemize}
\item Build unit tests for "{}your\+\_\+config"{}\+:
\begin{DoxyItemize}
\item cmake --build . --config your\+\_\+config
\end{DoxyItemize}
\item Run the unit tests for "{}your\+\_\+config"{} from the test folder\+:
\begin{DoxyItemize}
\item cd test \&\& ctest -\/C your\+\_\+config
\end{DoxyItemize}
\end{DoxyItemize}

Here is an example batch file. 
\begin{DoxyCode}{0}
\DoxyCodeLine{set\ build\_folder=out}
\DoxyCodeLine{mkdir\ \%build\_folder\%}
\DoxyCodeLine{pushd\ \%build\_folder\%}
\DoxyCodeLine{call\ cmake\ ..\ -\/DEASTL\_BUILD\_TESTS:BOOL=ON\ -\/DEASTL\_BUILD\_BENCHMARK:BOOL=OFF}
\DoxyCodeLine{call\ cmake\ -\/-\/build\ .\ -\/-\/config\ Release}
\DoxyCodeLine{call\ cmake\ -\/-\/build\ .\ -\/-\/config\ Debug}
\DoxyCodeLine{call\ cmake\ -\/-\/build\ .\ -\/-\/config\ RelWithDebInfo}
\DoxyCodeLine{call\ cmake\ -\/-\/build\ .\ -\/-\/config\ MinSizeRel}
\DoxyCodeLine{pushd\ test}
\DoxyCodeLine{call\ ctest\ -\/C\ Release}
\DoxyCodeLine{call\ ctest\ -\/C\ Debug}
\DoxyCodeLine{call\ ctest\ -\/C\ RelWithDebInfo}
\DoxyCodeLine{call\ ctest\ -\/C\ MinSizeRel}
\DoxyCodeLine{popd}
\DoxyCodeLine{popd}

\end{DoxyCode}
