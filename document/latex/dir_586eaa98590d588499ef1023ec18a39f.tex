\doxysubsection{详细描述}
The spine-\/ts runtime provides functionality to load and manipulate \href{http://esotericsoftware.com}{\texttt{ Spine}} skeletal animation data using Type\+Script and Java\+Script. spine-\/ts is split up into multiple modules\+:


\begin{DoxyEnumerate}
\item {\ttfamily spine-\/core/}, the core classes to load and process \doxylink{namespace_spine}{Spine} skeletons.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/webgl/}, a self-\/contained Web\+GL backend, built on the core classes.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/canvas/}, a self-\/contained Canvas backend, built on the core classes.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/canvaskit/}, a self-\/contained \href{https://skia.org/docs/user/modules/canvaskit/}{\texttt{ Canvas\+Kit}} backend, built on the core classes for Canvas\+Kit, supporting both Node\+JS for headless rendering, and browsers.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/threejs/}, a self-\/contained \href{https://threejs.org/}{\texttt{ THREE.\+JS}} backend, built on the core classes.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/player/}, a self-\/contained player to easily display \doxylink{namespace_spine}{Spine} animations on your website, built on the core classes and Web\+GL backend.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/phaser/}, a \href{https://phaser.io/}{\texttt{ Phaser}} backend, built on the core classes.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/pixi-\/v7/}, a \href{https://pixijs.com/}{\texttt{ Pixi\+JS v7}} backend, built on the core classes.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily spine-\/pixi-\/v8/}, a \href{https://pixijs.com/}{\texttt{ Pixi\+JS v8}} backend, built on the core classes.
\end{DoxyEnumerate}

In most cases, the {\ttfamily spine-\/player} module is best suited for your needs. Please refer to the \href{https://esotericsoftware.com/spine-player}{\texttt{ Spine Web Player documentation}} for more information.

For documentation of the core API in {\ttfamily spine-\/core}, please refer to our \href{http://esotericsoftware.com/spine-runtimes-guide}{\texttt{ Spine Runtimes Guide}}.

For documentation of {\ttfamily spine-\/phaser}, please refer to our \href{https://esotericsoftware.com/spine-phaser}{\texttt{ spine-\/phaser Guide}}.

For documentation of {\ttfamily spine-\/pixi-\/v7} and {\ttfamily spine-\/pixi-\/v8}, please refer to our \href{https://esotericsoftware.com/spine-pixi}{\texttt{ spine-\/pixi Guide}}.

For documentation of {\ttfamily spine-\/canvaskit}, please refer to our \href{https://esotericsoftware.com/spine-canvaskit}{\texttt{ spine-\/canvaskit Guide}}.

spine-\/ts works with data exported from \doxylink{namespace_spine}{Spine} 4.\+2.\+xx.

spine-\/ts Canvas does not support mesh attachments, clipping attachments, or two-\/color tinting. Only the alpha channel from tint colors is applied. Experimental support for mesh attachments can be enabled by setting {\ttfamily spine.\+Skeleton\+Renderer.\+use\+Triangle\+Rendering} to true. Note that this experimental mesh rendering is slow and render with artifacts on some browsers.

spine-\/canvaskit supports all \doxylink{namespace_spine}{Spine} features except two-\/color tinting.

The spine-\/webgl and spine-\/player support all \doxylink{namespace_spine}{Spine} features.

spine-\/ts THREE.\+JS does not support two color tinting. The THREE.\+JS backend provides {\ttfamily Skeleton\+Mesh.\+z\+Offset} to avoid z-\/fighting. Adjust to your near/far plane settings.\hypertarget{README.md_autotoc_md1088}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md1088}
