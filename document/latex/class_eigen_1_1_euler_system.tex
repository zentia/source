\subsubsection*{template$<$\mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}} \+\_\+\+Alpha\+Axis, \mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}} \+\_\+\+Beta\+Axis, \mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}} \+\_\+\+Gamma\+Axis$>$\newline
class Eigen\+::\+Euler\+System$<$ \+\_\+\+Alpha\+Axis, \+\_\+\+Beta\+Axis, \+\_\+\+Gamma\+Axis $>$}
Represents a fixed Euler rotation system. 

This meta-\/class goal is to represent the Euler system in compilation time, for \doxylink{class_eigen_1_1_euler_angles}{Euler\+Angles}.

You can use this class to get two things\+:
\begin{DoxyItemize}
\item Build an Euler system, and then pass it as a template parameter to \doxylink{class_eigen_1_1_euler_angles}{Euler\+Angles}.
\item Query some compile time data about an Euler system. (e.\+g. Whether it\textquotesingle{}s tait bryan)
\end{DoxyItemize}

Euler rotation is a set of three rotation on fixed axes. (see \doxylink{class_euler_angles}{Euler\+Angles}) This meta-\/class store constantly those signed axes. (see \doxylink{namespace_eigen_ae614aa7cdd687fb5c421a54f2ce5c361}{Euler\+Axis})\hypertarget{class_eigen_1_1_euler_system_autotoc_md1277}{}\doxysubsubsubsection{\texorpdfstring{Types of Euler systems}{Types of Euler systems}}\label{class_eigen_1_1_euler_system_autotoc_md1277}
All and only valid 3 dimension Euler rotation over standard signed axes\{+X,+Y,+Z,-\/X,-\/Y,-\/Z\} are supported\+:
\begin{DoxyItemize}
\item all axes X, Y, Z in each valid order (see below what order is valid)
\item rotation over the axis is supported both over the positive and negative directions.
\item both tait bryan and proper/classic Euler angles (i.\+e. the opposite).
\end{DoxyItemize}

Since \doxylink{class_eigen_1_1_euler_system}{Euler\+System} support both positive and negative directions, you may call this rotation distinction in other names\+:
\begin{DoxyItemize}
\item {\itshape right handed} or {\itshape left handed}
\item {\itshape counterclockwise} or {\itshape clockwise}
\end{DoxyItemize}

Notice all axed combination are valid, and would trigger a static assertion. Same unsigned axes can\textquotesingle{}t be neighbors, e.\+g. \{X,X,Y\} is invalid. This yield two and only two classes\+:
\begin{DoxyItemize}
\item {\itshape tait bryan} -\/ all unsigned axes are distinct, e.\+g. \{X,Y,Z\}
\item {\itshape proper/classic Euler angles} -\/ The first and the third unsigned axes is equal, and the second is different, e.\+g. \{X,Y,X\}
\end{DoxyItemize}\hypertarget{class_eigen_1_1_euler_system_autotoc_md1278}{}\doxysubsubsubsection{\texorpdfstring{Intrinsic vs extrinsic Euler systems}{Intrinsic vs extrinsic Euler systems}}\label{class_eigen_1_1_euler_system_autotoc_md1278}
Only intrinsic Euler systems are supported for simplicity. If you want to use extrinsic Euler systems, just use the equal intrinsic opposite order for axes and angles. I.\+e axes (\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A},\doxylink{bench__gemm_8cpp_a37a83060ac796961b44991c836f083f7}{B},C) becomes (C,\doxylink{bench__gemm_8cpp_a37a83060ac796961b44991c836f083f7}{B},\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A}), and angles (a,b,c) becomes (c,b,a).\hypertarget{class_eigen_1_1_euler_system_autotoc_md1279}{}\doxysubsubsubsection{\texorpdfstring{Convenient user typedefs}{Convenient user typedefs}}\label{class_eigen_1_1_euler_system_autotoc_md1279}
Convenient typedefs for \doxylink{class_eigen_1_1_euler_system}{Euler\+System} exist (only for positive axes Euler systems), in a form of \doxylink{class_eigen_1_1_euler_system}{Euler\+System}\{\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A}\}\{\doxylink{bench__gemm_8cpp_a37a83060ac796961b44991c836f083f7}{B}\}\{C\}, e.\+g. Euler\+System\+XYZ.\hypertarget{class_eigen_1_1_euler_system_autotoc_md1280}{}\doxysubsubsubsection{\texorpdfstring{Additional reading}{Additional reading}}\label{class_eigen_1_1_euler_system_autotoc_md1280}
