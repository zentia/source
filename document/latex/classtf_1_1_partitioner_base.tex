\subsubsection*{template$<$typename \mbox{\hyperlink{bench__gemm_8cpp_aaee470c3acb6b181fd804d072284fbaa}{C}} = Default\+Closure\+Wrapper$>$\newline
class tf\+::\+Partitioner\+Base$<$ C $>$}
class to derive a partitioner for scheduling parallel algorithms 


\begin{DoxyTemplParams}{模板参数}
{\em C} & closure wrapper type\\
\hline
\end{DoxyTemplParams}
The class provides base methods to derive a partitioner that can be used to schedule parallel iterations (e.\+g., \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each}).

An partitioner defines the scheduling method for running parallel algorithms, such \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each}, \doxylink{classtf_1_1_flow_builder_afb24798ebf46e253a40b01bffb1da6a7}{tf\+::\+Taskflow\+::reduce}, and so on. By default, we provide the following partitioners\+:


\begin{DoxyItemize}
\item \doxylink{classtf_1_1_guided_partitioner}{tf\+::\+Guided\+Partitioner} to enable guided scheduling algorithm of adaptive chunk size
\item \doxylink{classtf_1_1_dynamic_partitioner}{tf\+::\+Dynamic\+Partitioner} to enable dynamic scheduling algorithm of equal chunk size
\item \doxylink{classtf_1_1_static_partitioner}{tf\+::\+Static\+Partitioner} to enable static scheduling algorithm of static chunk size
\item \doxylink{classtf_1_1_random_partitioner}{tf\+::\+Random\+Partitioner} to enable random scheduling algorithm of random chunk size
\end{DoxyItemize}

Depending on applications, partitioning algorithms can impact the performance a lot. For example, if a parallel-\/iteration workload contains a regular work unit per iteration, \doxylink{classtf_1_1_static_partitioner}{tf\+::\+Static\+Partitioner} can deliver the best performance. On the other hand, if the work unit per iteration is irregular and unbalanced, \doxylink{classtf_1_1_guided_partitioner}{tf\+::\+Guided\+Partitioner} or \doxylink{classtf_1_1_dynamic_partitioner}{tf\+::\+Dynamic\+Partitioner} can outperform \doxylink{classtf_1_1_static_partitioner}{tf\+::\+Static\+Partitioner}. In most situations, \doxylink{classtf_1_1_guided_partitioner}{tf\+::\+Guided\+Partitioner} can deliver decent performance and is thus used as our default partitioner.

\begin{DoxyAttention}{注意}
Giving the partition size of 0 lets the Taskflow runtime automatically determines the partition size for the given partitioner.
\end{DoxyAttention}
In addition to partition size, the application can specify a closure wrapper for a partitioner. \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} closure wrapper allows the application to wrapper a partitioned task (i.\+e., closure) with a custom function object that performs additional tasks.
