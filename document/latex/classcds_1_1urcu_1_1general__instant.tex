class cds\+::urcu\+::general\+\_\+instant$<$ Lock, Backoff $>$}
User-\/space general-\/purpose RCU with immediate reclamation 

This is simplest general-\/purpose RCU implementation. When a thread calls \doxylink{classcds_1_1urcu_1_1general__instant_af2722e999c836a9b0e2c3322704753e2}{retire\+\_\+ptr} function the RCU {\ttfamily synchronize} function is called that waits until all reader/updater threads end up their read-\/side critical sections, i.\+e. until the RCU quiescent state will come. After that the retired object is freed immediately. Thus, the implementation blocks for any retired object

There is a wrapper \doxylink{classcds_1_1urcu_1_1gc_3_01general__instant_3_01_lock_00_01_backoff_01_4_01_4_cds_urcu_general_instant_gc}{gc\texorpdfstring{$<$}{<}general\+\_\+instant\texorpdfstring{$>$}{>}} for {\ttfamily general\+\_\+instant} class that provides unified RCU interface. You should use this wrapper class instead {\ttfamily general\+\_\+instant}

\item {\ttfamily Lock} -\/ mutex type, default is {\ttfamily std\+::mutex} 
\item {\ttfamily Backoff} -\/ back-\/off schema, default is \doxylink{namespacecds_1_1backoff_a0f64f7bc371e3ad006fba5bd855d6e96}{cds\+::backoff\+::\+Default} 
\end{DoxyItemize}
