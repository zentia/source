\doxysubsection{详细描述}
The spine-\/haxe runtime provides functionality to load, manipulate and render \href{http://esotericsoftware.com}{\texttt{ Spine}} skeletal animation data using \href{https://haxe.org/}{\texttt{ Haxe}} in combination with \href{https://www.openfl.org/}{\texttt{ Open\+FL}} and \href{https://lime.openfl.org/}{\texttt{ Lime}}.

For documentation of the core API in {\ttfamily spine-\/core}, please refer to our \href{http://esotericsoftware.com/spine-runtimes-guide}{\texttt{ Spine Runtimes Guide}}.

For documentation of {\ttfamily spine-\/haxe}, please refer to our \href{https://esotericsoftware.com/spine-haxe}{\texttt{ spine-\/haxe Guide}}.\hypertarget{README.md_autotoc_md1032}{}\doxysubsection{\texorpdfstring{Licensing}{Licensing}}\label{README.md_autotoc_md1032}

spine-\/haxe works with data exported from \doxylink{namespace_spine}{Spine} 4.\+2.\+xx.

spine-\/haxe supports all \doxylink{namespace_spine}{Spine} features except premultiplied alpha atlases and two color tinting.\hypertarget{README.md_autotoc_md1034}{}\doxysubsection{\texorpdfstring{Setup}{Setup}}\label{README.md_autotoc_md1034}
The spine-\/haxe runtime is composed of a core module, that is a Haxe implementation of the renderer-\/agnostic \doxylink{namespace_spine}{Spine} Runtimes core APIs, and the following specific renderer implementations\+:
\begin{DoxyItemize}
\item \href{https://lib.haxe.org/p/starling/}{\texttt{ Starling}}
\item \href{https://lib.haxe.org/p/flixel/}{\texttt{ Haxe\+Flixel}} (minimum supported version 5.\+9.\+0)
\end{DoxyItemize}

The core module of spine-\/haxe has zero dependencies. The rendering implementation depends on\+: openfl, starling, and flixel. To use spine-\/haxe you have first to install all the necessary dependencies\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{haxelib\ install\ openfl}
\DoxyCodeLine{haxelib\ install\ starling}
\DoxyCodeLine{haxelib\ install\ flixel}

\end{DoxyCode}


Once you have installed the dependencies, you can \href{https://esotericsoftware.com/files/spine-haxe/4.2/spine-haxe-latest.zip}{\texttt{ download the latest version of spine-\/haxe}} and install it\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{haxelib\ install\ spine-\/haxe-\/x.y.z.zip}

\end{DoxyCode}


Notice that the spine-\/haxe library is not available on \href{https://lib.haxe.org/}{\texttt{ lib.\+haxe.\+org}}. This is why you need to download the library and install it through the zip archive.\hypertarget{README.md_autotoc_md1035}{}\doxysubsection{\texorpdfstring{Example}{Example}}\label{README.md_autotoc_md1035}
The {\ttfamily example/} folder contains the spine-\/haxe examples. They demonstrates the usage of the API. Enter the the {\ttfamily spine-\/haxe} folder and run the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{lime\ test\ html5}

\end{DoxyCode}


This will compile the modules and start a server that serves the example pages at \href{http://127.0.0.1:3000}{\texttt{ http\+://127.\+0.\+0.\+1\+:3000}}.\hypertarget{README.md_autotoc_md1036}{}\doxysubsection{\texorpdfstring{Development}{Development}}\label{README.md_autotoc_md1036}
To setup the development environment install the following\+:


\begin{DoxyEnumerate}
\item \href{https://haxe.org/download/}{\texttt{ Haxe}}, ensure it\textquotesingle{}s available on the command line through your {\ttfamily PATH} if you use the binaries instead of the installer.
\item On the command line, run\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{haxelib\ setup}
\DoxyCodeLine{haxelib\ install\ openfl}
\DoxyCodeLine{haxelib\ run\ openfl\ setup}
\DoxyCodeLine{haxelib\ install\ starling}
\DoxyCodeLine{haxelib\ install\ flixel}

\end{DoxyCode}

\item Clone the {\ttfamily spine-\/runtimes} repository, and use {\ttfamily haxelib} to setup a dev library\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https://github.com/esotericsoftware/spine-\/runtimes}
\DoxyCodeLine{cd\ spine-\/runtimes}
\DoxyCodeLine{haxelib\ dev\ spine-\/haxe\ .}

\end{DoxyCode}

\end{DoxyEnumerate}

As an IDE, we recommend \href{https://code.visualstudio.com/}{\texttt{ Visual Studio Code}} with the following extension\+:


\begin{DoxyEnumerate}
\item \href{https://marketplace.visualstudio.com/items?itemName=nadako.vshaxe}{\texttt{ Haxe extension}}
\item \href{https://marketplace.visualstudio.com/items?itemName=vshaxe.hxcpp-debugger}{\texttt{ HXCPP debugger extension}}
\item \href{https://marketplace.visualstudio.com/items?itemName=openfl.lime-vscode-extension}{\texttt{ Lime extension}}
\end{DoxyEnumerate}

The extensions provide IDE features like auto-\/completion, debugging, and build support.

To debug a build, set the corresponding Lime target in the status bar at the bottom of VS Code to e.\+g. {\ttfamily HTML5 / Debug}. \doxylink{struct_run}{Run} the {\ttfamily lime} run configuration by pressing {\ttfamily F5}.
