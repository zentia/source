Michael\textquotesingle{}s hash set 

\label{classcds_1_1intrusive_1_1_michael_hash_set_cds_intrusive_MichaelHashSet_hp}%
\Hypertarget{classcds_1_1intrusive_1_1_michael_hash_set_cds_intrusive_MichaelHashSet_hp}%
 Source\+:
\begin{DoxyItemize}
\item \mbox{[}2002\mbox{]} Maged Michael "{}\+High performance dynamic lock-\/free hash tables and list-\/based sets"{}
\end{DoxyItemize}

Michael\textquotesingle{}s hash table algorithm is based on lock-\/free ordered list and it is very simple. The main structure is an array {\ttfamily T} of size {\ttfamily M}. Each element in {\ttfamily T} is basically a pointer to a hash bucket, implemented as a singly linked list. The array of buckets cannot be dynamically expanded. However, each bucket may contain unbounded number of items.

Template parameters are\+:
\begin{DoxyItemize}
\item {\ttfamily GC} -\/ Garbage collector used. Note the {\ttfamily GC} must be the same as the GC used for {\ttfamily Ordered\+List} 
\item {\ttfamily Ordered\+List} -\/ ordered list implementation used as bucket for hash set, possible implementations\+: {\ttfamily \doxylink{classcds_1_1intrusive_1_1_michael_list}{Michael\+List}}, {\ttfamily \doxylink{classcds_1_1intrusive_1_1_lazy_list}{Lazy\+List}}, {\ttfamily \doxylink{classcds_1_1intrusive_1_1_iterable_list}{Iterable\+List}}. The intrusive ordered list implementation specifies the type {\ttfamily T} stored in the hash-\/set, the reclamation schema {\ttfamily GC} used by hash-\/set, the comparison functor for the type {\ttfamily T} and other features specific for the ordered list.
\item {\ttfamily Traits} -\/ type traits. See {\ttfamily \doxylink{structcds_1_1intrusive_1_1michael__set_1_1traits}{michael\+\_\+set\+::traits}} for explanation. Instead of defining {\ttfamily Traits} struct you can use option-\/based syntax with {\ttfamily \doxylink{structcds_1_1intrusive_1_1michael__set_1_1make__traits}{michael\+\_\+set\+::make\+\_\+traits}} metafunction.
\end{DoxyItemize}

Forward iterator 

The forward iterator for Michael\textquotesingle{}s set is based on {\ttfamily Ordered\+List} forward iterator and has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item it iterates items in unordered fashion
\item The iterator cannot be moved across thread boundary because it may contain GC\textquotesingle{}s guard that is thread-\/private GC data.
\end{DoxyItemize}

Iterator thread safety depends on type of {\ttfamily Ordered\+List\+:} 
\begin{DoxyItemize}
\item for {\ttfamily \doxylink{classcds_1_1intrusive_1_1_michael_list}{Michael\+List}} and {\ttfamily \doxylink{classcds_1_1intrusive_1_1_lazy_list}{Lazy\+List}\+:} iterator guarantees safety even if you delete the item that iterator points to because that item is guarded by hazard pointer. However, in case of concurrent deleting operations it is no guarantee that you iterate all item in the set. Moreover, a crash is possible when you try to iterate the next element that has been deleted by concurrent thread. Use this iterator on the concurrent container for debugging purpose only.
\item for {\ttfamily \doxylink{classcds_1_1intrusive_1_1_iterable_list}{Iterable\+List}\+:} iterator is thread-\/safe. You may use it freely in concurrent environment. 
\end{DoxyItemize}\Hypertarget{classcds_1_1intrusive_1_1_michael_hash_set_a74df8e0e5de38581aaa233d6c8ef524c}\index{cds::intrusive::MichaelHashSet$<$ GC, OrderedList, Traits $>$@{cds::intrusive::MichaelHashSet$<$ GC, OrderedList, Traits $>$}!key\_comparator@{key\_comparator}}

Initializes hash set 

The Michael\textquotesingle{}s hash set is an unbounded container, but its hash table is non-\/expandable. At construction time you should pass estimated maximum item count and a load factor. The load factor is average size of one bucket -\/ a small number between 1 and 10. The bucket is an ordered single-\/linked list, searching in the bucket has linear complexity {\ttfamily O(n\+Load\+Factor)}. The constructor defines hash table size as rounding {\ttfamily n\+Max\+Item\+Count / n\+Load\+Factor} up to nearest power of two.

\hline
