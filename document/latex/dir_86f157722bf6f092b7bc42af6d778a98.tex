\doxysubsection{详细描述}
The spine-\/sdl runtime provides functionality to load, manipulate and render \href{http://esotericsoftware.com}{\texttt{ Spine}} skeletal animation data using \href{https://www.libsdl.org/}{\texttt{ SDL}}. spine-\/sdl is based on \href{../spine-c}{\texttt{ spine-\/c}} and \href{../spine-cpp}{\texttt{ spine-\/cpp}}, depending on whether you want to use a C or C++ implementation.\hypertarget{README.md_autotoc_md1059}{}\doxysubsection{\texorpdfstring{See the \href{http://esotericsoftware.com/spine-sdl}{\texttt{ spine-\/sdl documentation}} for in-\/depth information}{See the \href{http://esotericsoftware.com/spine-sdl}{\texttt{ spine-\/sdl documentation}} for in-\/depth information}}\label{README.md_autotoc_md1059}

spine-\/sdl works with data exported from \doxylink{namespace_spine}{Spine} 4.\+2.\+xx.

spine-\/sdl supports all \doxylink{namespace_spine}{Spine} features except screen blend mode and two color tinting.\hypertarget{README.md_autotoc_md1062}{}\doxysubsubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md1062}

\begin{DoxyEnumerate}
\item Create a new SDL project. See the \href{https://wiki.libsdl.org/FrontPage}{\texttt{ SDL documentation}} or have a look at the example in this repository.
\item Download the \doxylink{namespace_spine}{Spine} Runtimes source using git ({\ttfamily git clone \href{https://github.com/esotericsoftware/spine-runtimes}{\texttt{ https\+://github.\+com/esotericsoftware/spine-\/runtimes}}}) or download it as a zip via the download button above.
\item If you are using C, add the sources from {\ttfamily spine-\/c/spine-\/c/src/spine} and {\ttfamily \doxylink{spine-sdl-c_8c}{spine-\/sdl/src/spine-\/sdl-\/c.\+c}} to your project, and add the folder {\ttfamily spine-\/c/spine-\/c/include} and {\ttfamily \doxylink{spine-sdl-c_8h}{spine-\/sdl/src/spine-\/sdl-\/c.\+h}} to your header search path. Note that includes are specified as {\ttfamily \#inclue \texorpdfstring{$<$}{<}spine/file.\+h\texorpdfstring{$>$}{>}}, so the {\ttfamily spine} directory cannot be omitted when copying the source files.
\item If you are using C++, add the sources from {\ttfamily spine-\/cpp/spine-\/cpp/src/spine} and {\ttfamily \doxylink{spine-sdl-cpp_8cpp}{spine-\/sdl/src/spine-\/sdl-\/cpp.\+cpp}} to your project, and add the folder {\ttfamily spine-\/cpp/spine-\/cpp/include} and {\ttfamily \doxylink{spine-sdl-cpp_8h}{spine-\/sdl/src/spine-\/sdl-\/cpp.\+h}} to your header search path. Note that includes are specified as {\ttfamily \#include \texorpdfstring{$<$}{<}spine/file.\+h\texorpdfstring{$>$}{>}}, so the {\ttfamily spine} directory cannot be omitted when copying the source files.
\end{DoxyEnumerate}

See the \href{http://esotericsoftware.com/spine-documentation\#runtimesTitle}{\texttt{ Spine Runtimes documentation}} on how to use the APIs or check out the \doxylink{namespace_spine}{Spine} SDL example in this repository.\hypertarget{README.md_autotoc_md1063}{}\doxysubsubsection{\texorpdfstring{Example}{Example}}\label{README.md_autotoc_md1063}
The \doxylink{namespace_spine}{Spine} SDL example works on Windows, Linux and Mac OS X. For a spine-\/c based example, see \href{example/main.c}{\texttt{ example/main.\+c}}, for a spine-\/cpp example see \href{example/main.cpp}{\texttt{ example/main.\+cpp}}.\hypertarget{README.md_autotoc_md1064}{}\doxysubsubsubsection{\texorpdfstring{Windows}{Windows}}\label{README.md_autotoc_md1064}

\begin{DoxyEnumerate}
\item Install \href{https://www.visualstudio.com/en-us/downloads/download-visual-studio-vs.aspx}{\texttt{ Visual Studio Community}}. Make sure you install support for C++ as well as th Windows SDK for XP/7/8.
\item Install CMake via the \href{https://cmake.org/download/}{\texttt{ Windows installer package}}.
\item Download the \doxylink{namespace_spine}{Spine} Runtimes repository using git ({\ttfamily git clone \href{https://github.com/esotericsoftware/spine-runtimes}{\texttt{ https\+://github.\+com/esotericsoftware/spine-\/runtimes}}}) or download it as a zip via the download button above.
\item \doxylink{struct_run}{Run} CMake GUI from the start menu
\item Click {\ttfamily Browse Source} and select the directory {\ttfamily spine-\/runtimes}
\item Click {\ttfamily Browse Build} and select the {\ttfamily spine-\/runtimes/spine-\/sdl/build} directory. You can create the {\ttfamily build} folder directly in the file dialog via {\ttfamily New Folder}.
\item Click {\ttfamily Configure}. Then click {\ttfamily Generate}. This will create a Visual Studio solution file called {\ttfamily spine.\+sln} in {\ttfamily spine-\/runtimes/spine-\/sdl/build} and also download the SDL dependencies.
\item Open the {\ttfamily spine.\+sln} file in Visual Studio
\item Right click the {\ttfamily spine-\/sdl-\/example-\/c} or {\ttfamily spine-\/sdl-\/example-\/cpp} project in the solution explorer and select {\ttfamily Set as Startup Project} from the context menus
\item Click {\ttfamily Local Windows Debugger} to run the example
\end{DoxyEnumerate}
