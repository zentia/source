Load \doxylink{struct_t_b_b}{TBB} at runtime. 

{\bfseries{Usage\+:}}

Link with {\ttfamily tbbproxy.\+lib} (or {\ttfamily libtbbproxy.\+a}) instead of {\ttfamily tbb.\+lib} ({\ttfamily libtbb.\+dylib}, {\ttfamily libtbb.\+so}).

\doxylink{struct_t_b_b}{TBB} library will be loaded at runtime from {\ttfamily } $<$install dir$>$/lib/ia32 directory.

{\bfseries{Attention\+:}} 

All {\ttfamily \doxylink{classtbb_1_1interface6_1_1runtime__loader}{runtime\+\_\+loader}} objects (in the same module, i.\+e. exe or dll) share some global state. The most noticeable piece of global state is loaded \doxylink{struct_t_b_b}{TBB} library. There are some implications\+: \begin{DoxyVerb}-   Only one TBB library can be loaded per module.

-   If one object has already loaded TBB library, another object will not load TBB.
    If the loaded TBB library is suitable for the second object, both will use TBB
    cooperatively, otherwise the second object will report an error.

-   \c runtime_loader objects will not work (correctly) in parallel due to absence of
    synchronization.
\end{DoxyVerb}

Load \doxylink{struct_t_b_b}{TBB}. 

The method searches the directories specified in {\ttfamily path}\mbox{[}\mbox{]} array for the \doxylink{struct_t_b_b}{TBB} library. When the library is found, it is loaded and its version is checked. If the version is not suitable, the library is unloaded, and the search continues.

{\bfseries{Note\+:}} 

For security reasons, avoid using relative directory names. For example, never load \doxylink{struct_t_b_b}{TBB} from current ({\ttfamily "{}."{}}), parent ({\ttfamily "{}.."{}}) or any other relative directory (like {\ttfamily "{}lib"{}} ). Use only absolute directory names (e. g. "{}/usr/local/lib"{}).

For the same security reasons, avoid using system default directories ({\ttfamily "{}"{}}) on Windows. (See \href{http://www.microsoft.com/technet/security/advisory/2269637.mspx}{\texttt{ http\+://www.\+microsoft.\+com/technet/security/advisory/2269637.\+mspx}} for details.)

Neglecting these rules may cause your program to execute 3-\/rd party malicious code.
