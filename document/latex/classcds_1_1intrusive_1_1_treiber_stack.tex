Treiber intrusive stack 

Intrusive implementation of well-\/known Treiber\textquotesingle{}s stack algorithm\+:
\begin{DoxyItemize}
\item R. K. Treiber. Systems programming\+: Coping with parallelism. Technical Report RJ 5118, IBM Almaden Research Center, April 1986.
\end{DoxyItemize}

\doxylink{namespacecds_1_1algo_1_1elimination_cds_elimination_description}{Elimination back-\/off technique} can be used optionally. The idea of elimination algorithm is taken from\+:
\begin{DoxyItemize}
\item \mbox{[}2004\mbox{]} Danny Hendler, Nir Shavit, Lena Yerushalmi "{}\+A Scalable Lock-\/free Stack Algorithm"{}
\end{DoxyItemize}

The elimination algorithm uses a single elimination array as a back-\/off schema on a shared lock-\/free stack. If the threads fail on the stack, they attempt to eliminate on the array, and if they fail in eliminating, they attempt to access the stack again and so on.

\begin{DoxyNote}{注解}
Hendler\textquotesingle{}s et al paper describes a lock-\/free implementation of elimination back-\/off which is quite complex. The main difficulty is the managing life-\/time of elimination record. This implementation uses simplified lock-\/based (spin-\/based) approach which allows the elimination record allocation on thread\textquotesingle{}s stack. This approach demonstrates sufficient performance under high load.
\end{DoxyNote}
