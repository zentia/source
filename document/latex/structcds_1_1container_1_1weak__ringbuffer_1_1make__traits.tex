Metafunction converting option list to {\ttfamily \doxylink{structcds_1_1container_1_1weak__ringbuffer_1_1traits}{weak\+\_\+ringbuffer\+::traits}} 

Supported {\ttfamily Options} are\+:
\begin{DoxyItemize}
\item {\ttfamily \doxylink{structcds_1_1opt_1_1buffer}{opt\+::buffer}} -\/ an uninitialized buffer type for internal cyclic array. Possible types are\+: {\ttfamily \doxylink{classcds_1_1opt_1_1v_1_1uninitialized__dynamic__buffer}{opt\+::v\+::uninitialized\+\_\+dynamic\+\_\+buffer}} (the default), {\ttfamily \doxylink{classcds_1_1opt_1_1v_1_1uninitialized__static__buffer}{opt\+::v\+::uninitialized\+\_\+static\+\_\+buffer}}. The type of element in the buffer is not important\+: it will be changed via {\ttfamily rebind} metafunction.
\item {\ttfamily \doxylink{structcds_1_1opt_1_1value__cleaner}{opt\+::value\+\_\+cleaner}} -\/ a functor to clean items dequeued. The functor calls the destructor for ring-\/buffer item. After a set of items is dequeued, {\ttfamily value\+\_\+cleaner} cleans the cells that the items have been occupied. If {\ttfamily T} is a complex type, {\ttfamily value\+\_\+cleaner} can be an useful feature. Default value is \doxylink{structcds_1_1opt_1_1v_1_1empty__cleaner}{opt\+::v\+::empty\+\_\+cleaner} that is suitable for POD types.
\item {\ttfamily \doxylink{structcds_1_1opt_1_1padding}{opt\+::padding}} -\/ padding for internal critical atomic data. Default is {\ttfamily \doxylink{namespacecds_1_1opt_a3e664c94f4a1373ec267d19df5cc3beaad2c97dd99643d7a2c379cc7e0750028a}{opt\+::cache\+\_\+line\+\_\+padding}} 
\item {\ttfamily \doxylink{structcds_1_1opt_1_1memory__model}{opt\+::memory\+\_\+model}} -\/ C++ memory ordering model. Can be {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1relaxed__ordering}{opt\+::v\+::relaxed\+\_\+ordering}} (relaxed memory model, the default) or {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1sequential__consistent}{opt\+::v\+::sequential\+\_\+consistent}} (sequentially consisnent memory model).
\end{DoxyItemize}
