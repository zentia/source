Updates the value for {\ttfamily key} 

The operation performs inserting or updating the value for {\ttfamily key} with lock-\/free manner. If {\ttfamily b\+Insert} is {\ttfamily false}, only updating of existing node is possible.

If {\ttfamily key} is not found and inserting is allowed (i.\+e. {\ttfamily b\+Insert} is {\ttfamily true}), then the new node created from {\ttfamily key} will be inserted into the map; note that in this case the \doxylink{classcds_1_1container_1_1_bronson_a_v_l_tree_map_3_01cds_1_1urcu_1_1gc_3_01_r_c_u_01_4_00_01_key_00_01_t_01_5_00_01_traits_01_4_af1241636918651c696267269ff61e682}{key\+\_\+type} should be constructible from type {\ttfamily K}. Otherwise, the value for {\ttfamily key} will be changed to {\ttfamily p\+Val}.

RCU {\ttfamily synchronize()} method can be called. RCU should not be locked.

Returns {\ttfamily  std\+::pair$<$bool, bool$>$ } where {\ttfamily first} is {\ttfamily true} if operation is successful, {\ttfamily second} is {\ttfamily true} if new node has been added or {\ttfamily false} if the node with {\ttfamily key} already exists.
