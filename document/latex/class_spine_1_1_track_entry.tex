Resets the rotation directions for mixing this entry\textquotesingle{}s rotate timelines. This can be useful to avoid bones rotating the long way around when using \doxylink{class_spine_1_1_track_entry_ae28ac3337847d6dd81b8062c9bb09b57}{alpha} and starting animations on other tracks.

The two rotations likely change over time, so which direction is the short or long way also changes. If true, when mixing from the previous animation to this animation, the previous animation is applied as normal instead of being mixed out.

When mixing between animations that key the same property, if a lower track also keys that property then the value will briefly dip toward the lower track value during the mix. \mbox{\hyperlink{sugar_8h_a0d954e435550ebcb2035013bfea2db85}{float}} Spine.\+Track\+Entry.\+Track\+End\hspace{0.3cm}{\ttfamily [get]}, {\ttfamily [set]}}

The track time in seconds when this animation will be removed from the track. Defaults to the highest possible float value, meaning the animation will be applied until a new animation is set or the track is cleared. If the track end time is reached, no other animations are queued for playback, and mixing from any previous animations is complete, then the properties keyed by the animation are set to the setup pose and the track is cleared.

It may be desired to use \doxylink{class_spine_1_1_animation_state_ad66c903cef5ad7fa9eee840342d7388a}{Animation\+State.\+Add\+Empty\+Animation(int, float, float)} rather than have the animation abruptly cease being applied.\Hypertarget{class_spine_1_1_track_entry_a8ff80753d9c6b2cdb8d0933492581e55}\index{Spine.TrackEntry@{Spine.TrackEntry}!TrackIndex@{TrackIndex}}
