class tf\+::\+Data\+Pipe$<$ Input, Output, C $>$}
class to create a stage in a data-\/parallel pipeline 

\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} data pipe represents a stage of a data-\/parallel pipeline. \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} data pipe can be either {\itshape parallel} direction or {\itshape serial} direction (specified by \doxylink{namespacetf_abb7a11e41fd457f69e7ff45d4c769564}{tf\+::\+Pipe\+Type}) and is associated with a callable to invoke by the pipeline scheduler.

You need to use the template function, \doxylink{namespacetf_a8975fa5762088789adb0b60f38208309}{tf\+::make\+\_\+data\+\_\+pipe}, to create a data pipe. The input and output types of a \doxylink{classtf_1_1_data_pipe}{tf\+::\+Data\+Pipe} should be decayed types (though the library will always decay them for you using {\ttfamily std\+::decay}) to allow internal storage to work. The data will be passed by reference to your callable, at which you can take it by copy or reference.
