\subsubsection*{template$<$typename Iterator$>$\newline
class eastl\+::move\+\_\+iterator$<$ Iterator $>$}
unwrap\+\_\+reverse\+\_\+iterator is not implemented since there\textquotesingle{}s no good use case and there\textquotesingle{}s some abiguitiy. Note that unwrap\+\_\+iterator(reverse\+\_\+iterator$<$\+T$>$) returns \doxylink{classeastl_1_1reverse__iterator}{reverse\+\_\+iterator$<$unwrap(\+T)$>$}. However, given what unwrap\+\_\+generic\+\_\+iterator and unwrap\+\_\+move\+\_\+iterator do, one might expect unwrap\+\_\+reverse\+\_\+iterator(reverse\+\_\+iterator$<$\+T$>$) to return T, which is not the same. To avoid that confusion, and because there\textquotesingle{}s no current use case for this, we don\textquotesingle{}t provide unwrap\+\_\+reverse\+\_\+iterator. \doxylink{classeastl_1_1move__iterator}{move\+\_\+iterator}

From the C++11 Standard, section 24.\+5.\+3.\+1\+: Class template \doxylink{classeastl_1_1move__iterator}{move\+\_\+iterator} is an iterator adaptor with the same behavior as the underlying iterator except that its dereference operator implicitly converts the value returned by the underlying iterator\textquotesingle{}s dereference operator to an rvalue reference. Some generic algorithms can be called with move iterators to replace copying with moving.
