\doxysubsection{Porting from Eigen2 to Eigen3}
\hypertarget{_eigen2_to_eigen3}{}\label{_eigen2_to_eigen3}\index{Porting from Eigen2 to Eigen3@{Porting from Eigen2 to Eigen3}}
This page lists the most important API changes between Eigen2 and Eigen3, and gives tips to help porting your application from Eigen2 to Eigen3.

\textbackslash{}eigen\+Auto\+Toc\hypertarget{_eigen2_to_eigen3_CompatibilitySupport}{}\doxysubsubsection{\texorpdfstring{Eigen2 compatibility support}{Eigen2 compatibility support}}\label{_eigen2_to_eigen3_CompatibilitySupport}
Up to version 3.\+2 Eigen provides \href{http://eigen.tuxfamily.org/dox-3.2/Eigen2SupportModes.html}{\texttt{ Eigen2 support modes}}. These are removed now, because they were barely used anymore and became hard to maintain after internal re-\/designs. You can still use them by first \href{http://eigen.tuxfamily.org/dox-3.2/Eigen2ToEigen3.html}{\texttt{ porting your code to Eigen 3.\+2}}.\hypertarget{_eigen2_to_eigen3_Using}{}\doxysubsubsection{\texorpdfstring{The USING\+\_\+\+PART\+\_\+\+OF\+\_\+\+NAMESPACE\+\_\+\+EIGEN macro}{The USING\+\_\+\+PART\+\_\+\+OF\+\_\+\+NAMESPACE\+\_\+\+EIGEN macro}}\label{_eigen2_to_eigen3_Using}
The USING\+\_\+\+PART\+\_\+\+OF\+\_\+\+NAMESPACE\+\_\+\+EIGEN macro has been removed. In \doxylink{namespace_eigen}{Eigen} 3, just do\+:

\hypertarget{_eigen2_to_eigen3_ComplexDot}{}\doxysubsubsection{\texorpdfstring{Dot products over complex numbers}{Dot products over complex numbers}}\label{_eigen2_to_eigen3_ComplexDot}
This is the single trickiest change between \doxylink{namespace_eigen}{Eigen} 2 and \doxylink{namespace_eigen}{Eigen} 3. It only affects code using {\ttfamily std\+::complex} numbers as scalar type.

\doxylink{namespace_eigen}{Eigen} 2\textquotesingle{}s dot product was linear in the first variable. \doxylink{namespace_eigen}{Eigen} 3\textquotesingle{}s dot product is linear in the second variable. In other words, the \doxylink{namespace_eigen}{Eigen} 2 code 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{offscreen_8c_a5fd331c99e778f04762be6d8173eb4d2}{x}}.dot(\mbox{\hyperlink{namespace_eigen_ae998329a5bd49a32b6048a27e1a232a0}{y}})\ }

\end{DoxyCode}
 is equivalent to the \doxylink{namespace_eigen}{Eigen} 3 code 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{namespace_eigen_ae998329a5bd49a32b6048a27e1a232a0}{y}}.dot(\mbox{\hyperlink{offscreen_8c_a5fd331c99e778f04762be6d8173eb4d2}{x}})\ }

\end{DoxyCode}
 In yet other words, dot products are complex-\/conjugated in \doxylink{namespace_eigen}{Eigen} 3 compared to \doxylink{namespace_eigen}{Eigen} 2. See in class \doxylink{class_eigen_1_1_dense_base}{Dense\+Base}.\hypertarget{_eigen2_to_eigen3_CoefficientWiseOperations}{}\doxysubsubsection{\texorpdfstring{Coefficient wise operations}{Coefficient wise operations}}\label{_eigen2_to_eigen3_CoefficientWiseOperations}
In Eigen2, coefficient wise operations which have no proper mathematical definition (as a coefficient wise product) were achieved using the .cwise() prefix, e.\+g.\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{_cwise__product_8cpp_ad2cbe4616e813eb9af81732dca777b24}{a}}.cwise()\ *\ \mbox{\hyperlink{offscreen_8c_a846c9667e34d56c560bb7f0ac6e173f6}{b}}\ }

\end{DoxyCode}
 In Eigen3 this .cwise() prefix has been superseded by a new kind of matrix type called \doxylink{class_eigen_1_1_array}{Array} for which all operations are performed coefficient wise. You can easily view a matrix as an array and vice versa using the \doxylink{class_eigen_1_1_matrix_base_a6bd01a3c94daf864579f14510aa320f8}{Matrix\+Base\+::array()} and \doxylink{class_eigen_1_1_array_base_a059ebdf0bcc64c26d4a314919f929a95}{Array\+Base\+::matrix()} functions respectively. Here is an example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Vector4f\ \mbox{\hyperlink{_cwise__product_8cpp_ad2cbe4616e813eb9af81732dca777b24}{a}},\ \mbox{\hyperlink{offscreen_8c_a846c9667e34d56c560bb7f0ac6e173f6}{b}},\ \mbox{\hyperlink{bench_vec_add_8cpp_a41689956983587b085f9da3e48f31d99}{c}};}

\end{DoxyCode}
 Note that the .\doxylink{sugar_8h_af6228a2253f88b2affbe0c54d8b5b321}{array()} function is not at all a synonym of the deprecated .cwise() prefix. While the .cwise() prefix changed the behavior of the following operator, the \doxylink{sugar_8h_af6228a2253f88b2affbe0c54d8b5b321}{array()} function performs a permanent conversion to the array world. Therefore, for binary operations such as the coefficient wise product, both sides must be converted to an {\itshape array} as in the above example. \hypertarget{_eigen2_to_eigen3_LazyVsNoalias}{}\doxysubsubsection{\texorpdfstring{Lazy evaluation and noalias}{Lazy evaluation and noalias}}\label{_eigen2_to_eigen3_LazyVsNoalias}
In \doxylink{namespace_eigen}{Eigen} all operations are performed in a lazy fashion except the matrix products which are always evaluated into a temporary by default. In Eigen2, lazy evaluation could be enforced by tagging a product using the .lazy() function. However, in complex expressions it was not easy to determine where to put the lazy() function. In Eigen3, the lazy() feature has been superseded by the \doxylink{class_eigen_1_1_matrix_base_a2c1085de7645f23f240876388457da0b}{Matrix\+Base\+::noalias()} function which can be used on the left hand side of an assignment when no aliasing can occur. \hypertarget{_eigen2_to_eigen3_eiPrefix}{}\doxysubsubsection{\texorpdfstring{Internal ei\+\_\+ prefix}{Internal ei\+\_\+ prefix}}\label{_eigen2_to_eigen3_eiPrefix}
In Eigen2, global internal functions and structures were prefixed by {\ttfamily ei\+\_\+}. In Eigen3, they all have been moved into the more explicit {\ttfamily internal} namespace.
