multimap

Implements a canonical multimap.

The large majority of the implementation of this class is found in the rbtree base class. We control the behaviour of rbtree via template parameters.

Pool allocation If you want to make a custom memory pool for a multimap container, your pool needs to contain items of type \doxylink{classeastl_1_1multimap_a4e64fdf1283ed14b1ef27702d412e4a0}{multimap\+::node\+\_\+type}. So if you have a memory pool that has a constructor that takes the size of pool items and the count of pool items, you would do this (assuming that Memory\+Pool implements the Allocator interface)\+: typedef multimap\texorpdfstring{$<$}{<}Widget, int, \doxylink{structeastl_1_1less}{less$<$\+Widget$>$}, Memory\+Pool\texorpdfstring{$>$}{>} Widget\+Map; // Delare your Widget\+Map type. Memory\+Pool my\+Pool(sizeof(\+Widget\+Map\+::node\+\_\+type), 100); // Make a pool of 100 Widget nodes. Widget\+Map my\+Map(\&my\+Pool); // Create a map that uses the pool.
