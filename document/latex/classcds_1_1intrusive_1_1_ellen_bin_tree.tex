class cds\+::intrusive\+::\+Ellen\+Bin\+Tree$<$ GC, Key, T, Traits $>$}
Ellen\textquotesingle{}s et al binary search tree 

\label{classcds_1_1intrusive_1_1_ellen_bin_tree_cds_intrusive_EllenBinTree}%
\Hypertarget{classcds_1_1intrusive_1_1_ellen_bin_tree_cds_intrusive_EllenBinTree}%
 Source\+:
\begin{DoxyItemize}
\item \mbox{[}2010\mbox{]} F.\+Ellen, P.\+Fatourou, E.\+Ruppert, F.\+van Breugel "{}\+Non-\/blocking Binary Search Tree"{}
\end{DoxyItemize}

Ellen\+Bin\+Tree is an {\itshape unbalanced} leaf-\/oriented binary search tree that implements the {\itshape set} abstract data type. Nodes maintains child pointers but not parent pointers. Every internal node has exactly two children, and all data of type {\ttfamily T} currently in the tree are stored in the leaves. Internal nodes of the tree are used to direct {\ttfamily \doxylink{classcds_1_1intrusive_1_1_ellen_bin_tree_a2e58a04f767c910f217531738d876e6b}{find()}} operation along the path to the correct leaf. The keys (of {\ttfamily Key} type) stored in internal nodes may or may not be in the set. {\ttfamily Key} type is a subset of {\ttfamily T} type. There should be exactly defined a key extracting functor for converting object of type {\ttfamily T} to object of type {\ttfamily Key}.

Due to {\ttfamily \doxylink{classcds_1_1intrusive_1_1_ellen_bin_tree_a7b4b540395597d0a296ef01fd15568f3}{extract\+\_\+min()}} and {\ttfamily \doxylink{classcds_1_1intrusive_1_1_ellen_bin_tree_a66fef3313f95b887854de03dd7032140}{extract\+\_\+max()}} member functions the {\ttfamily Ellen\+Bin\+Tree} can act as a {\itshape priority queue}. In this case you should provide unique compound key, for example, the priority value plus some uniformly distributed random value.

\begin{DoxyNote}{注解}
In the current implementation we do not use helping technique described in the original paper. In Hazard Pointer schema the helping is too complicated and does not give any observable benefits. Instead of helping, when a thread encounters a concurrent operation it just spins waiting for the operation done. Such solution allows greatly simplify implementation of the tree.
\end{DoxyNote}
\begin{DoxyAttention}{注意}
Recall the tree is {\bfseries{unbalanced}}. The complexity of operations is {\ttfamily O(log N)} for uniformly distributed random keys, but in the worst case the complexity is {\ttfamily O(\+N)}.
\end{DoxyAttention}
