\subsubsection*{template$<$typename T, typename Deleter = smart\+\_\+ptr\+\_\+deleter$<$\+T$>$$>$\newline
class eastl\+::scoped\+\_\+ptr$<$ T, Deleter $>$}
class \doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr}

This class is intended to be the same as the C++11 \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr} class, but was created before there was such a thing.

This class implements a \doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} template. This is a class which is similar to the C++ auto\+\_\+ptr template, except that it prohibits copying of itself, for safety.

More specifically, the \doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} class template stores a pointer to a dynamically allocated object. The object pointed to is automatically deleted on destructor of \doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} or can be manually deleted via the scopted\+\_\+ptr\+::reset function.

\doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} cannot be used in C++ Standard Library containers; you\textquotesingle{}ll need to use the \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr} template if you want to do this. The reason you can\textquotesingle{}t use \doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} is that it prohibits copying. You can\textquotesingle{}t (safely) use auto\+\_\+ptr in C++ Standard Library containers because copying of an auto\+\_\+ptr will create a situation whereby objects are multiply freed.

\doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} cannot be used with arrays of objects. The reason for this is that it calls delete on the owned pointer and not delete\mbox{[}\mbox{]}. The latter allows for the calling of the destructors for the objects of the owned pointer. If you want to use \doxylink{classeastl_1_1scoped__ptr}{scoped\+\_\+ptr} with a dynamically allocated array, use the \doxylink{classeastl_1_1scoped__array}{scoped\+\_\+array} function instead.
