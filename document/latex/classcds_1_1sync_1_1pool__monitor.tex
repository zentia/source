class cds\+::sync\+::pool\+\_\+monitor$<$ Lock\+Pool, Back\+Off, Stat $>$}
\doxysectlink{cds_sync_monitor}{Monitor}{0} that allocates node\textquotesingle{}s lock when needed 

The monitor is intended for reducing the number of system mutexes for huge containers like a tree. The monitor allocates the mutex from the pool {\ttfamily Lock\+Pool} only when container\textquotesingle{}s node should be locked. Lifetime of node\textquotesingle{}s mutex is managed by reference counter. When the reference counter to node\textquotesingle{}s mutex becomes zero, the mutex is given back to the pool.

The monitor is blocked\+: the access to node\textquotesingle{}s mutex is performed under the spin-\/lock. However, node locking/unlocking is performed beyond the spin-\/lock.

Template arguments\+:
\begin{DoxyItemize}
\item {\ttfamily Lock\+Pool} -\/ the \doxylink{group__cds__memory__pool}{pool type}. The pool must maintain the objects of type {\ttfamily std\+::mutex} or similar. The access to the pool is not synchronized.
\item {\ttfamily Back\+Off} -\/ back-\/off strategy for spinning, default is {\ttfamily \doxylink{namespacecds_1_1backoff_a0f64f7bc371e3ad006fba5bd855d6e96}{cds\+::backoff\+::\+Default}} 
\item {\ttfamily Stat} -\/ enable ({\ttfamily true}) or disable ({\ttfamily false}, the default) monitor\textquotesingle{}s internal statistics.
\end{DoxyItemize}
