\subsubsection*{template$<$typename T, \mbox{\hyperlink{sugar_8h_abb452686968e48b67397da5f97445f5b}{bool}} auto\+Construct = true, \mbox{\hyperlink{sugar_8h_abb452686968e48b67397da5f97445f5b}{bool}} auto\+Destruct = true$>$\newline
class eastl\+::late\+\_\+constructed$<$ T, auto\+Construct, auto\+Destruct $>$}
\doxylink{classeastl_1_1late__constructed}{late\+\_\+constructed}

Implements a smart pointer type which separates the memory allocation of an object from the object\textquotesingle{}s construction. The primary use case is to declare a global variable of the late\+\_\+construction type, which allows the memory to be global but the constructor executes at some point after \doxylink{main-override-static_8c_ae66f6b31b5ad750f1fe042a706a4e3d4}{main()} begins as opposed to before main, which is often dangerous for non-\/trivial types.

The auto\+Construct template parameter controls whether the object is automatically default constructed upon first reference or must be manually constructed upon the first use of operator \texorpdfstring{$\ast$}{*} or -\/\texorpdfstring{$>$}{>}. auto\+Construct is convenient but it causes \texorpdfstring{$\ast$}{*} and -\/\texorpdfstring{$>$}{>} to be slightly slower and may result in construction at an inconvenient time.

The auto\+Destruct template parameter controls whether the object, if constructed, is automatically destructed when \doxylink{classeastl_1_1late__constructed_af645bdfa61183ec1b4dd9cbd847c4bd7}{\texorpdfstring{$\sim$}{\string~}late\+\_\+constructed()} is called or must be manually destructed via a call to \doxylink{classeastl_1_1late__constructed_a14d4bd685b9b8b506a22b342907fcee7}{destruct()}.

While construction can be automatic or manual, automatic destruction support is always present. Thus you aren\textquotesingle{}t required in any case to manually call destruct. However, you may safely manually destruct the object at any time before the \doxylink{classeastl_1_1late__constructed}{late\+\_\+constructed} destructor is executed.

You may still use \doxylink{classeastl_1_1late__constructed}{late\+\_\+constructed} after calling \doxylink{classeastl_1_1late__constructed_a14d4bd685b9b8b506a22b342907fcee7}{destruct()}, including calling \doxylink{classeastl_1_1late__constructed_ad66fb9bcf9fdcf49fa20912ea710d893}{construct()} again to reconstruct the instance. destruct returns the \doxylink{classeastl_1_1late__constructed}{late\+\_\+constructed} instance to a state equivalent to before construct was called.

Caveat\+: While \doxylink{classeastl_1_1late__constructed}{late\+\_\+constructed} instances can be declared in global scope and initialize prior to \doxylink{main-override-static_8c_ae66f6b31b5ad750f1fe042a706a4e3d4}{main()} executing, you cannot otherwise use such globally declared instances prior to main with guaranteed behavior unless you can ensure that the \doxylink{classeastl_1_1late__constructed}{late\+\_\+constructed} instance is itself constructed prior to your use of it.

Example usage (demonstrating manual-\/construction)\+: late\+\_\+constructed$<$\+Widget, false$>$ g\+Widget;

void \doxylink{main-override-static_8c_ae66f6b31b5ad750f1fe042a706a4e3d4}{main()}\{ g\+Widget.\+construct(k\+Scrollbar\+Type, k\+Vertical, "{}\+My\+Scrollbar"{}); g\+Widget-\/\texorpdfstring{$>$}{>}Set\+Value(15); g\+Widget.\+destruct(); \}

Example usage (demonstrating auto-\/construction)\+: late\+\_\+constructed$<$\+Widget, true$>$ g\+Widget;

void \doxylink{main-override-static_8c_ae66f6b31b5ad750f1fe042a706a4e3d4}{main()}\{ g\+Widget-\/\texorpdfstring{$>$}{>}Set\+Value(15); // You may want to call destruct here, but aren\textquotesingle{}t required to do so unless the Widget type requires it. \}
