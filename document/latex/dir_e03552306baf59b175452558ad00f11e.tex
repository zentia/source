\doxysubsection{详细描述}
This experimental plugin provides integration of UI Toolkit for the spine-\/unity runtime. Please be sure to test this package first and create backups of your project before using. The plugin adds a {\ttfamily Spine\+Visual\+Element} control element to be used with UI Toolkit, providing functionality similar to {\ttfamily Skeleton\+Graphic} to add your \doxylink{namespace_spine}{Spine} skeletons to a Visual \doxylink{class_tree}{Tree} Asset via UI Toolkit\textquotesingle{}s UI Builder.\hypertarget{README.md_autotoc_md1141}{}\doxysubsubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md1141}
For usage of the {\ttfamily Spine\+Visual\+Element} control element, you can check out the included {\ttfamily UI Toolkit Example.\+unity} example scene and {\ttfamily Spine\+UIToolkit.\+uxml} file which can be installed via the {\ttfamily Samples} tab in the Unity Package Manager window. The new {\ttfamily Spine\+Visual\+Element} can be found in the Unity {\ttfamily UI Builder} window under {\ttfamily Library -\/ Project -\/ Custom Controls} {\ttfamily \doxylink{namespace_spine}{Spine} -\/ Unity -\/ Spine\+Visual\+Element}. It can be added to your hierarchy and configured as normal UI Toolkit elements while providing \doxylink{namespace_spine}{Spine} skeleton attributes as well.\hypertarget{README.md_autotoc_md1142}{}\doxysubsubsection{\texorpdfstring{Limitations}{Limitations}}\label{README.md_autotoc_md1142}
Premultiply-\/alpha (PMA) atlas textures are not supported due to limitations of the UI Toolkit {\ttfamily Visual\+Element} API, not allowing custom shaders. Please export your atlas textures as straight alpha to avoid artifacts.

Custom slot blend modes (additive, muliply, screen) are not supported due to the same custom shader limitation.
