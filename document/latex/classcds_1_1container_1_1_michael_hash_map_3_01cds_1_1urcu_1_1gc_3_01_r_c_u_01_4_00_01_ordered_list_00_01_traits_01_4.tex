Michael\textquotesingle{}s hash table algorithm is based on lock-\/free ordered list and it is very simple. The main structure is an array {\ttfamily T} of size {\ttfamily M}. Each element in {\ttfamily T} is basically a pointer to a hash bucket, implemented as a singly linked list. The array of buckets cannot be dynamically expanded. However, each bucket may contain unbounded number of items.

Template parameters are\+:
\begin{DoxyItemize}
\item {\ttfamily RCU} -\/ one of \doxylink{namespacecds_1_1urcu_cds_urcu_gc}{RCU type}
\item {\ttfamily Ordered\+List} -\/ ordered key-\/value list implementation used as bucket for hash map, for example, {\ttfamily \doxylink{classcds_1_1container_1_1_michael_k_v_list}{Michael\+KVList}}. The ordered list implementation specifies the {\ttfamily Key} and {\ttfamily \doxylink{struct_value}{Value}} types stored in the hash-\/map, the reclamation schema {\ttfamily GC} used by hash-\/map, the comparison functor for the type {\ttfamily Key} and other features specific for the ordered list.
\item {\ttfamily Traits} -\/ map traits, default is {\ttfamily \doxylink{namespacecds_1_1container_1_1michael__map_ab5f19c5329fdd2d525e9c82b5c741539}{michael\+\_\+map\+::traits}}. Instead of defining {\ttfamily Traits} struct you may use option-\/based syntax with {\ttfamily \doxylink{namespacecds_1_1container_1_1michael__map_ab675a965e57c642c90f4fd01ae2adfa8}{michael\+\_\+map\+::make\+\_\+traits}} metafunction
\end{DoxyItemize}

Many of the class function take a key argument of type {\ttfamily K} that in general is not {\ttfamily \doxylink{classcds_1_1container_1_1_michael_hash_map_3_01cds_1_1urcu_1_1gc_3_01_r_c_u_01_4_00_01_ordered_list_00_01_traits_01_4_a7475912bb124d87f03b47a441c781d45}{key\+\_\+type}}. {\ttfamily \doxylink{classcds_1_1container_1_1_michael_hash_map_3_01cds_1_1urcu_1_1gc_3_01_r_c_u_01_4_00_01_ordered_list_00_01_traits_01_4_a7475912bb124d87f03b47a441c781d45}{key\+\_\+type}} and an argument of template type {\ttfamily K} must meet the following requirements\+:
\begin{DoxyItemize}

Forward iterator 

The forward iterator for Michael\textquotesingle{}s map has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item to protect the value, the iterator contains a GC-\/specific guard + another guard is required locally for increment operator. For some GC (like as {\ttfamily \doxylink{namespacecds_1_1gc_af0523881f089593a7559d194ee2b5dd9}{gc\+::\+HP}}), a guard is a limited resource per thread, so an exception (or assertion) "{}no free guard"{} may be thrown if the limit of guard count per thread is exceeded.
\item The iterator cannot be moved across thread boundary because it contains thread-\/private GC\textquotesingle{}s guard.
\end{DoxyItemize}

Iterator thread safety depends on type of {\ttfamily Ordered\+List\+:} 
\begin{DoxyItemize}
\item for {\ttfamily \doxylink{classcds_1_1container_1_1_michael_k_v_list}{Michael\+KVList}} and {\ttfamily \doxylink{classcds_1_1container_1_1_lazy_k_v_list}{Lazy\+KVList}\+:} iterator guarantees safety even if you delete the item that iterator points to because that item is guarded by hazard pointer. However, in case of concurrent deleting operations it is no guarantee that you iterate all item in the map. Moreover, a crash is possible when you try to iterate the next element that has been deleted by concurrent thread. Use this iterator on the concurrent container for debugging purpose only.
\item for {\ttfamily \doxylink{classcds_1_1container_1_1_iterable_list}{Iterable\+List}\+:} iterator is thread-\/safe. You may use it freely in concurrent environment.
\end{DoxyItemize}

The Michael\textquotesingle{}s hash map is non-\/expandable container. You should point the average count of items {\ttfamily n\+Max\+Item\+Count} when you create an object. {\ttfamily n\+Load\+Factor} parameter defines average count of items per bucket and it should be small number between 1 and 10. Remember, since the bucket implementation is an ordered list, searching in the bucket is linear \mbox{[}{\ttfamily O(n\+Load\+Factor)}\mbox{]}. Note, that many popular STL hash map implementation uses load factor 1.

The ctor defines hash table size as rounding {\ttfamily n\+Mac\+Item\+Count / n\+Load\+Factor} up to nearest power of two.
