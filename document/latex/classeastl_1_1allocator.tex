\doxysubsection{详细描述}
allocator

In this allocator class, note that it is not templated on any type and instead it simply allocates blocks of memory much like the C malloc and free functions. It can be thought of as similar to C++ std\+::allocator$<$char$>$. The flags parameter has meaning that is specific to the allocation

C++11\textquotesingle{}s std\+::allocator (20.\+6.\+9) doesn\textquotesingle{}t have a move constructor or assignment operator. This is possibly because std\+::allocators are associated with types instead of as instances. The potential non-\/equivalance of C++ std\+::allocator instances has been a source of some acknowledged design problems. We don\textquotesingle{}t implement support for move construction or assignment in \doxylink{classeastl_1_1allocator}{eastl\+::allocator}, but users can define their own allocators which do have move functions and the eastl containers are compatible with such allocators (i.\+e. nothing unexpected will happen).
