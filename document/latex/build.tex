\chapter{Building}
\hypertarget{build}{}\label{build}\index{Building@{Building}}
Checkout the sources from Git\+Hub\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https:\textcolor{comment}{//github.com/microsoft/mimalloc}}

\end{DoxyCode}
\hypertarget{build_autotoc_md19}{}\doxysubsection{\texorpdfstring{Windows}{Windows}}\label{build_autotoc_md19}
Open {\ttfamily ide/vs2019/mimalloc.\+sln} in Visual Studio 2019 and build (or {\ttfamily ide/vs2017/mimalloc.\+sln}). The {\ttfamily mimalloc} project builds a static library (in {\ttfamily out/msvc-\/x64}), while the {\ttfamily mimalloc-\/override} project builds a DLL for overriding malloc in the entire program.\hypertarget{build_autotoc_md20}{}\doxysubsection{\texorpdfstring{mac\+OS, Linux, BSD, etc.}{mac\+OS, Linux, BSD, etc.}}\label{build_autotoc_md20}
We use \href{https://cmake.org}{\texttt{ {\ttfamily cmake}}}\textsuperscript{1} as the build system\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{>\ mkdir\ -\/\mbox{\hyperlink{main-override_8cpp_a117104b82864d3b23ec174af6d392709}{p}}\ out/release}
\DoxyCodeLine{>\ cd\ out/release}
\DoxyCodeLine{>\ cmake\ ../..}
\DoxyCodeLine{>\ make}

\end{DoxyCode}
 This builds the library as a shared (dynamic) library ({\ttfamily .so} or {\ttfamily .dylib}), a static library ({\ttfamily .a}), and as a single object file ({\ttfamily .o}).

{\ttfamily \texorpdfstring{$>$}{>} sudo make install} (install the library and header files in {\ttfamily /usr/local/lib} and {\ttfamily /usr/local/include})

You can build the debug version which does many internal checks and maintains detailed statistics as\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{>\ mkdir\ -\/\mbox{\hyperlink{main-override_8cpp_a117104b82864d3b23ec174af6d392709}{p}}\ out/\mbox{\hyperlink{base_8h_adfa139c9a1902753b3696794759b2c48aad42f6697b035b7580e4fef93be20b4d}{debug}}}
\DoxyCodeLine{>\ cd\ out/\mbox{\hyperlink{base_8h_adfa139c9a1902753b3696794759b2c48aad42f6697b035b7580e4fef93be20b4d}{debug}}}
\DoxyCodeLine{>\ cmake\ -\/DCMAKE\_BUILD\_TYPE=Debug\ ../..}
\DoxyCodeLine{>\ make}

\end{DoxyCode}
 This will name the shared library as {\ttfamily libmimalloc-\/debug.\+so}.

Finally, you can build a {\itshape secure} version that uses guard pages, encrypted free lists, etc, as\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{>\ mkdir\ -\/\mbox{\hyperlink{main-override_8cpp_a117104b82864d3b23ec174af6d392709}{p}}\ out/secure}
\DoxyCodeLine{>\ cd\ out/secure}
\DoxyCodeLine{>\ cmake\ -\/DMI\_SECURE=ON\ ../..}
\DoxyCodeLine{>\ make}

\end{DoxyCode}
 This will name the shared library as {\ttfamily libmimalloc-\/secure.\+so}. Use {\ttfamily ccmake}\textsuperscript{2} instead of {\ttfamily cmake} to see and customize all the available build options.

Notes\+:
\begin{DoxyEnumerate}
\item Install CMake\+: {\ttfamily sudo apt-\/get install cmake}
\item Install CCMake\+: {\ttfamily sudo apt-\/get install cmake-\/curses-\/gui} 
\end{DoxyEnumerate}
