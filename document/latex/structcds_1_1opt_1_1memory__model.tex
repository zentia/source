\mbox{[}type-\/option\mbox{]} Option setter for C++ memory model 

The {\bfseries{cds}} library supports following memory ordering constraints for atomic operations in container implementation\+:
\begin{DoxyItemize}
\item {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1relaxed__ordering}{v\+::relaxed\+\_\+ordering}} -\/ relaxed C++ memory model. This mode supports full set of memory ordering constraints\+: {\ttfamily memory\+\_\+order\+\_\+relaxed}, {\ttfamily memory\+\_\+order\+\_\+acquire}, {\ttfamily memory\+\_\+order\+\_\+release} and so on.
\item {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1sequential__consistent}{v\+::sequential\+\_\+consistent}} -\/ sequentially consistent C++ memory model (default memory ordering for C++). In this mode any memory ordering constraint maps to {\ttfamily memory\+\_\+order\+\_\+seq\+\_\+cst}.
\end{DoxyItemize}

The {\ttfamily \doxylink{ir__common_8h_a1ac75f8fee198af783521627f1fecfbc}{Type}} template parameter can be {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1relaxed__ordering}{v\+::relaxed\+\_\+ordering}} or {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1sequential__consistent}{v\+::sequential\+\_\+consistent}}.

You may mix different memory ordering options for different containers\+: one declare as sequentially consistent, another declare as relaxed. Usually, {\ttfamily \doxylink{structcds_1_1opt_1_1v_1_1relaxed__ordering}{v\+::relaxed\+\_\+ordering}} is the default memory ordering for {\bfseries{libcds}} containers.
