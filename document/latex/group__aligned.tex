\doxysubsection{详细描述}
Allocating aligned memory blocks. Note that {\ttfamily alignment} always follows {\ttfamily size} for consistency with the unaligned allocation API, but unfortunately this differs from {\ttfamily posix\+\_\+memalign} and {\ttfamily aligned\+\_\+alloc} in the C library.

\mbox{\hyperlink{mimalloc_8h_a9d6d8aef94ac19034a5f163606f84830}{void}} \texorpdfstring{$\ast$}{*} mi\+\_\+malloc\+\_\+aligned (\begin{DoxyParamCaption}\item[{size\+\_\+t}]{size}{, }\item[{size\+\_\+t}]{alignment}{}\end{DoxyParamCaption})}

Allocate {\itshape size} bytes aligned by {\itshape alignment}. 
\begin{DoxyParams}{参数}
{\em size} & number of bytes to allocate. \\
\hline
{\em alignment} & the minimal alignment of the allocated memory. \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{返回}
pointer to the allocated memory or {\itshape NULL} if out of memory, or if the alignment is not a power of 2 (including 0). The {\itshape size} is unrestricted (and does not have to be an integral multiple of the {\itshape alignment}). The returned pointer is aligned by {\itshape alignment}, i.\+e. {\ttfamily (uintptr\+\_\+t)p \% alignment == 0}. Returns a unique pointer if called with {\itshape size} 0.
\end{DoxyReturn}
Note that {\ttfamily alignment} always follows {\ttfamily size} for consistency with the unaligned allocation API, but unfortunately this differs from {\ttfamily posix\+\_\+memalign} and {\ttfamily aligned\+\_\+alloc} in the C library.
