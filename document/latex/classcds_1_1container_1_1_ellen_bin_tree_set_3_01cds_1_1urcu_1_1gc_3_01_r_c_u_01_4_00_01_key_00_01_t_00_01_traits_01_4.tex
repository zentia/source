class cds\+::container\+::\+Ellen\+Bin\+Tree\+Set$<$ cds\+::urcu\+::gc$<$ RCU $>$, Key, T, Traits $>$}
Set based on Ellen\textquotesingle{}s et al binary search tree (RCU specialization) 

\label{classcds_1_1container_1_1_ellen_bin_tree_set_3_01cds_1_1urcu_1_1gc_3_01_r_c_u_01_4_00_01_key_00_01_t_00_01_traits_01_4_cds_container_EllenBinTreeSet_rcu}%
\Hypertarget{classcds_1_1container_1_1_ellen_bin_tree_set_3_01cds_1_1urcu_1_1gc_3_01_r_c_u_01_4_00_01_key_00_01_t_00_01_traits_01_4_cds_container_EllenBinTreeSet_rcu}%
 Source\+:
\begin{DoxyItemize}
\item \mbox{[}2010\mbox{]} F.\+Ellen, P.\+Fatourou, E.\+Ruppert, F.\+van Breugel "{}\+Non-\/blocking Binary Search Tree"{}
\end{DoxyItemize}

Ellen\+Bin\+Tree\+Set is an unbalanced leaf-\/oriented binary search tree that implements the {\itshape set} abstract data type. Nodes maintains child pointers but not parent pointers. Every internal node has exactly two children, and all data of type {\ttfamily T} currently in the tree are stored in the leaves. Internal nodes of the tree are used to direct {\ttfamily find} operation along the path to the correct leaf. The keys (of {\ttfamily Key} type) stored in internal nodes may or may not be in the set. {\ttfamily Key} type is a subset of {\ttfamily T} type. There should be exactly defined a key extracting functor for converting object of type {\ttfamily T} to object of type {\ttfamily Key}.

Due to {\ttfamily extract\+\_\+min} and {\ttfamily extract\+\_\+max} member functions the {\ttfamily Ellen\+Bin\+Tree\+Set} can act as a {\itshape priority queue}. In this case you should provide unique compound key, for example, the priority value plus some uniformly distributed random value.

\begin{DoxyWarning}{警告}
Recall the tree is {\bfseries{unbalanced}}. The complexity of operations is {\ttfamily O(log N)} for uniformly distributed random keys, but in the worst case the complexity is {\ttfamily O(\+N)}.
\end{DoxyWarning}
\begin{DoxyNote}{注解}
In the current implementation we do not use helping technique described in original paper. So, the current implementation is near to fine-\/grained lock-\/based tree. Helping will be implemented in future release
\end{DoxyNote}
