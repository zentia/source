\label{classcds_1_1container_1_1_lazy_list_cds_nonintrusive_LazyList_gc}%
\Hypertarget{classcds_1_1container_1_1_lazy_list_cds_nonintrusive_LazyList_gc}%
 Usually, ordered single-\/linked list is used as a building block for the hash table implementation. The complexity of searching is {\ttfamily O(\+N)}.

Source\+:
\begin{DoxyItemize}
\item \mbox{[}2005\mbox{]} Steve Heller, Maurice Herlihy, Victor Luchangco, Mark Moir, William N. Scherer III, and Nir Shavit "{}\+A Lazy Concurrent List-\/\+Based Set Algorithm"{}
\end{DoxyItemize}

The lazy list is based on an optimistic locking scheme for inserts and removes, eliminating the need to use the equivalent of an atomically markable reference. It also has a novel wait-\/free membership {\ttfamily \doxylink{classcds_1_1container_1_1_lazy_list_ac9ff4a62f58cdf783878a5f5bc39f051}{find()}} operation that does not need to perform cleanup operations and is more efficient.

It is non-\/intrusive version of {\ttfamily \doxylink{classcds_1_1intrusive_1_1_lazy_list}{cds\+::intrusive\+::\+Lazy\+List}} class.

Unlike standard container, this implementation does not divide type {\ttfamily T} into key and value part and may be used as main building block for hash set algorithms.

The key is a function (or a part) of type {\ttfamily T}, and the comparing function is specified by {\ttfamily Traits\+::compare} functor or {\ttfamily Traits\+::less} predicate.

{\ttfamily \doxylink{classcds_1_1container_1_1_lazy_k_v_list}{Lazy\+KVList}} is a key-\/value version of lazy non-\/intrusive list that is closer to the C++ std library approach.

Forward iterator 

The forward iterator for lazy list has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item to protect the value, the iterator contains a GC-\/specific guard + another guard is required locally for increment operator. For some GC ({\ttfamily \doxylink{namespacecds_1_1gc_af0523881f089593a7559d194ee2b5dd9}{gc\+::\+HP}}), a guard is limited resource per thread, so an exception (or assertion) "{}no free guard"{} may be thrown if a limit of guard count per thread is exceeded.
\item The iterator cannot be moved across thread boundary since it contains GC\textquotesingle{}s guard that is thread-\/private GC data.
\item Iterator ensures thread-\/safety even if you delete the item that iterator points to. However, in case of concurrent deleting operations it is no guarantee that you iterate all item in the list. Moreover, a crash is possible when you try to iterate the next element that has been deleted by concurrent thread.
\end{DoxyItemize}

\begin{DoxyWarning}{警告}
Use this iterator on the concurrent container for debugging purpose only. 
\end{DoxyWarning}

The argument {\ttfamily item} of user-\/defined functor {\ttfamily func} is the reference to the list\textquotesingle{}s item inserted. When {\ttfamily func} is called it has exclusive access to the item. The user-\/defined functor is called only if the inserting is success.

The type {\ttfamily Q} should contain the complete key of the node. The object of {\ttfamily \doxylink{classcds_1_1container_1_1_lazy_list_a93402d80970262e4395e3d783545bf5e}{value\+\_\+type}} should be constructible from {\ttfamily key} of type {\ttfamily Q}.

The function allows to split creating of new item into two part\+:
\begin{DoxyItemize}
\item create item from {\ttfamily key} with initializing key-\/fields only;
\item insert new item into the list;
