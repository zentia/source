Aligned type 

This meta-\/algorithm solves compiler problem when you need to declare a type {\ttfamily T} with alignment equal to another type alignment. For example, the following declaration produces an error in Microsoft Visual Studio 2008 compiler\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{typedef}\ \textcolor{keywordtype}{double}\ \ my\_double;}
\DoxyCodeLine{\textcolor{keyword}{typedef}\ \mbox{\hyperlink{etw_8h_aa2cfaf13e297017861f3769e3eb87cae}{\_\_declspec}}(\mbox{\hyperlink{runtime_2spdlog_2include_2spdlog_2fmt_2bundled_2base_8h_a6c52c4bac77f6744fdc338766e9e8a6f}{align}}(\ \_\_alignof(my\_double)))\ \textcolor{keywordtype}{int}\ \ \ aligned\_int;}

\end{DoxyCode}
 In MS VS, the \+\_\+\+\_\+declspec(align(\+N)) construction requires that N must be a integer constant (1, 2, 4 and so on) but not an integer constant expression.

The result of this meta-\/algo is a type {\ttfamily aligned\+\_\+type$<$\+T,\+Alignment$>$\+::type} that is {\ttfamily T} aligned by {\ttfamily Alignment}. For example, with {\ttfamily \doxylink{structcds_1_1details_1_1aligned__type}{aligned\+\_\+type}} the prevoius example will not generate an error\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{typedef}\ \textcolor{keywordtype}{double}\ \ my\_double;}
\DoxyCodeLine{\textcolor{keyword}{typedef}\ \textcolor{keyword}{typename}\ cds::details::aligned\_type<\mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}},\ \_\_alignof(my\_double)>\mbox{\hyperlink{imgui__impl__opengl3__loader_8h_ae8396ff85d63082521e3324820df1009}{::type}}\ \ \ aligned\_int;}

\end{DoxyCode}
 and result of this declaration is equivalent to 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{typedef}\ \mbox{\hyperlink{etw_8h_aa2cfaf13e297017861f3769e3eb87cae}{\_\_declspec}}(\mbox{\hyperlink{runtime_2spdlog_2include_2spdlog_2fmt_2bundled_2base_8h_a6c52c4bac77f6744fdc338766e9e8a6f}{align}}(8))\ \mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}}\ \ \ aligned\_int;}

\end{DoxyCode}


The {\ttfamily Alignment} template parameter must be a constant expression and its result must be power of two. The maximum of its value is 1024.
