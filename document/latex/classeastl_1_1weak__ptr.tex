\subsubsection*{template$<$typename T$>$\newline
class eastl\+::weak\+\_\+ptr$<$ T $>$}
\doxylink{classeastl_1_1weak__ptr}{weak\+\_\+ptr}

The \doxylink{classeastl_1_1weak__ptr}{weak\+\_\+ptr} class template stores a "{}weak reference"{} to an object that\textquotesingle{}s already managed by a \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr}. To access the object, a \doxylink{classeastl_1_1weak__ptr}{weak\+\_\+ptr} can be converted to a \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr} using the \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr} constructor or the \doxylink{classeastl_1_1weak__ptr_a4cc40ab5a5bdccabda3e1e7097a665cf}{lock()} member function. When the last \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr} to the object goes away and the object is deleted, the attempt to obtain a \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr} from the \doxylink{classeastl_1_1weak__ptr}{weak\+\_\+ptr} instances that refer to the deleted object will fail via \doxylink{classeastl_1_1weak__ptr_a4cc40ab5a5bdccabda3e1e7097a665cf}{lock()} returning an empty \doxylink{classeastl_1_1shared__ptr}{shared\+\_\+ptr}.

The Allocator template argument manages the memory of the shared reference count and not the stored object. \doxylink{classeastl_1_1weak__ptr}{weak\+\_\+ptr} will not delete the stored object but instead can only delete the reference count on that object.
