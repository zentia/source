class eastl\+::fixed\+\_\+list$<$ T, node\+Count, b\+Enable\+Overflow, Overflow\+Allocator $>$}
\doxylink{classeastl_1_1fixed__list}{fixed\+\_\+list}

\doxylink{classeastl_1_1fixed__list}{fixed\+\_\+list} is a list which uses a single block of contiguous memory for its nodes. The purpose of this is to reduce memory usage relative to a conventional memory allocation system (with block headers), to increase allocation speed (often due to avoidance of mutex locks), to increase performance (due to better memory locality), and to decrease memory fragmentation due to the way that fixed block allocators work.

The primary downside to a \doxylink{classeastl_1_1fixed__list}{fixed\+\_\+list} is that the number of nodes it can contain is fixed upon its declaration. If you want a \doxylink{classeastl_1_1fixed__list}{fixed\+\_\+list} that doesn\textquotesingle{}t have this limitation, then you probably don\textquotesingle{}t want a \doxylink{classeastl_1_1fixed__list}{fixed\+\_\+list}. You can always create your own memory allocator that works the way you want.

Template parameters\+: T The type of object the list holds. node\+Count The max number of objects to contain. b\+Enable\+Overflow Whether or not we should use the overflow heap if our object pool is exhausted. Overflow\+Allocator Overflow allocator, which is only used if b\+Enable\+Overflow == true. Defaults to the global heap.
