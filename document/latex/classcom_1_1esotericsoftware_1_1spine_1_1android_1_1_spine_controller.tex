\doxysubsection{详细描述}
Controls how the skeleton of a {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_view}{Spine\+View}} is animated and rendered.

Upon initialization of a {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_view}{Spine\+View}}, the provided {\ttfamily on\+Initialized} callback method is called once. This method can be used to set up the initial animation(s) of the skeleton, among other things.

After initialization is complete, the {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_view}{Spine\+View}} is rendered at the screen refresh rate. In each frame, the {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_animation_state}{Animation\+State}} is updated and applied to the {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_skeleton}{Skeleton}}.

Next, the optionally provided method {\ttfamily on\+Before\+Update\+World\+Transforms} is called, which can modify the skeleton before its current pose is calculated using {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_skeleton_a9bd5945b87acb026a03b5fd32e8171c1}{Skeleton\#update\+World\+Transform(\+Skeleton.\+Physics)}}. After {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_skeleton_a9bd5945b87acb026a03b5fd32e8171c1}{Skeleton\#update\+World\+Transform(\+Skeleton.\+Physics)}} has completed, the optional {\ttfamily on\+After\+Update\+World\+Transforms} method is called, which can modify the current pose before rendering the skeleton.

Before the skeleton\textquotesingle{}s current pose is rendered by the {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_view}{Spine\+View}}, the optional {\ttfamily on\+Before\+Paint} is called, which allows rendering backgrounds or other objects that should go behind the skeleton on the {\ttfamily \doxylink{}{Canvas}}. The {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_view}{Spine\+View}} then renders the skeleton\textquotesingle{}s current pose and finally calls the optional {\ttfamily on\+After\+Paint}, which can render additional objects on top of the skeleton.

The underlying {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_android_texture_atlas}{Android\+Texture\+Atlas}}, {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_skeleton_data}{Skeleton\+Data}}, {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_skeleton}{Skeleton}}, {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_animation_state_data}{Animation\+State\+Data}}, {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_animation_state}{Animation\+State}}, and {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_android_skeleton_drawable}{Android\+Skeleton\+Drawable}} can be accessed through their respective getters to inspect and/or modify the skeleton and its associated data. Accessing this data is only allowed if the {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_view}{Spine\+View}} and its data have been initialized and have not been disposed of yet.

By default, the widget updates and renders the skeleton every frame. The {\ttfamily pause} method can be used to pause updating and rendering the skeleton. The {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_controller_a184dbb1de962e6fdb27cc9fdcf15beb3}{Spine\+Controller\#resume()}} method resumes updating and rendering the skeleton. The {\ttfamily \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1android_1_1_spine_controller_a014e7a179f54ea510e69acc640df62d1}{Spine\+Controller\#is\+Playing()}} getter reports the current state.
