\chapter{xxhsum(1) -- print or check xx\+Hash non-\/cryptographic checksums}
\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81}{}\label{md_runtime_2xx_hash_2cli_2xxhsum_81}\index{xxhsum(1) -- print or check xxHash non-\/cryptographic checksums@{xxhsum(1) -- print or check xxHash non-\/cryptographic checksums}}
\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md378}{}\doxysection{\texorpdfstring{SYNOPSIS}{SYNOPSIS}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md378}
{\ttfamily xxhsum} \mbox{[}{\itshape OPTION}\mbox{]}... \mbox{[}{\itshape FILE}\mbox{]}...

{\ttfamily xxhsum -\/b} \mbox{[}{\itshape OPTION}\mbox{]}...

{\ttfamily xxh32sum} is equivalent to {\ttfamily xxhsum -\/H0}, {\ttfamily xxh64sum} is equivalent to {\ttfamily xxhsum -\/H1}, {\ttfamily xxh128sum} is equivalent to {\ttfamily xxhsum -\/H2}, {\ttfamily xxh3sum} is equivalent to {\ttfamily xxhsum -\/H3}.\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md379}{}\doxysection{\texorpdfstring{DESCRIPTION}{DESCRIPTION}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md379}
Print or check xx\+Hash (32, 64 or 128 bits) checksums. When no {\itshape FILE}, read standard input, except if it\textquotesingle{}s the console. When {\itshape FILE} is {\ttfamily -\/}, read standard input even if it\textquotesingle{}s the console.

{\ttfamily xxhsum} supports a command line syntax similar but not identical to md5sum(1). Differences are\+:


\begin{DoxyItemize}
\item {\ttfamily xxhsum} doesn\textquotesingle{}t have text mode switch ({\ttfamily -\/t})
\item {\ttfamily xxhsum} doesn\textquotesingle{}t have short binary mode switch ({\ttfamily -\/b})
\item {\ttfamily xxhsum} always treats files as binary file
\item {\ttfamily xxhsum} has a hash selection switch ({\ttfamily -\/H})
\end{DoxyItemize}

As xx\+Hash is a fast non-\/cryptographic checksum algorithm, {\ttfamily xxhsum} should not be used for security related purposes.

{\ttfamily xxhsum -\/b} invokes benchmark mode. See OPTIONS and EXAMPLES for details.\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md380}{}\doxysection{\texorpdfstring{OPTIONS}{OPTIONS}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md380}

\begin{DoxyItemize}
\item {\ttfamily -\/H}\texorpdfstring{$\ast$}{*}\+HASHTYPE\texorpdfstring{$\ast$}{*}\+: Hash selection. {\itshape HASHTYPE} means {\ttfamily 0}=XXH32, {\ttfamily 1}=XXH64, {\ttfamily 2}=XXH128, {\ttfamily 3}=XXH3. Alternatively, {\itshape HASHTYPE} {\ttfamily 32}=XXH32, {\ttfamily 64}=XXH64, {\ttfamily 128}=XXH128. Default value is {\ttfamily 1} (XXH64)
\item {\ttfamily -\/-\/binary}\+: Read in binary mode.
\item {\ttfamily -\/-\/tag}\+: Output in the BSD style.
\item {\ttfamily -\/-\/little-\/endian}\+: Set output hexadecimal checksum value as little endian convention. By default, value is displayed as big endian.
\item {\ttfamily -\/V}, {\ttfamily -\/-\/version}\+: Displays xxhsum version and exits
\item {\ttfamily -\/h}, {\ttfamily -\/-\/help}\+: Displays help and exits
\end{DoxyItemize}\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md381}{}\doxysubsection{\texorpdfstring{Advanced file input options}{Advanced file input options}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md381}

\begin{DoxyItemize}
\item {\ttfamily -\/-\/files-\/from}, {\ttfamily -\/-\/filelist} {\itshape FILE}\+: Read filenames from {\itshape FILE} and generate hashes for them. {\ttfamily stdin} is also a valid way to provide filenames (when no {\itshape FILE} or {\ttfamily -\/} provided). Valid format is one filename per line, which can include embedded spaces, etc with no need for quotes, escapes, etc. A line commencing with \textquotesingle{}\textbackslash{}\textquotesingle{} will enable the convention used in the encoding of filenames against output hashes, whereby subsequent \textbackslash{}\textbackslash{}, ~\newline
 and \textbackslash{}r seqeuences are converted to the single character 0x5C, 0x0A and 0x0D respectively.
\end{DoxyItemize}\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md382}{}\doxysubsection{\texorpdfstring{The following options are useful only for checksum verification\+:}{The following options are useful only for checksum verification\+:}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md382}

\begin{DoxyItemize}
\item {\ttfamily -\/c}, {\ttfamily -\/-\/check} {\itshape FILE}\+: Read xx\+Hash sums from {\itshape FILE} and check them
\item {\ttfamily -\/-\/strict}\+: Return an error code if any line in {\itshape FILE} is invalid, not just if some checksums are wrong. This policy is disabled by default, though UI will prompt an informational message if any line in the file is detected invalid.
\item {\ttfamily -\/w}, {\ttfamily -\/-\/warn}\+: Emit a warning message about each improperly formatted line in {\itshape FILE}.
\item {\ttfamily -\/q}, {\ttfamily -\/-\/quiet}\+: Don\textquotesingle{}t print OK for each successfully verified hash
\item {\ttfamily -\/-\/status}\+: Don\textquotesingle{}t output anything. Only generate a Status code to show success.
\end{DoxyItemize}\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md383}{}\doxysubsection{\texorpdfstring{The following options are useful only benchmark purpose\+:}{The following options are useful only benchmark purpose\+:}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md383}

\begin{DoxyItemize}
\item {\ttfamily -\/b}\+: Benchmark mode. See EXAMPLES for details.
\item {\ttfamily -\/b\#}\+: Specify ID of variant to be tested. Multiple variants can be selected, separated by a \textquotesingle{},\textquotesingle{} comma.
\item {\ttfamily -\/B}\texorpdfstring{$\ast$}{*}\+BLOCKSIZE\texorpdfstring{$\ast$}{*}\+: Only useful for benchmark mode ({\ttfamily -\/b}). See {\itshape EXAMPLES} for details. $<$\+BLOCKSIZE$>$ specifies benchmark mode\textquotesingle{}s test data block size in bytes. Default value is 102400
\item {\ttfamily -\/i}\texorpdfstring{$\ast$}{*}\+ITERATIONS\texorpdfstring{$\ast$}{*}\+: Only useful for benchmark mode ({\ttfamily -\/b}). See {\itshape EXAMPLES} for details. $<$\+ITERATIONS$>$ specifies number of iterations in benchmark. Single iteration lasts approximately 1000 milliseconds. Default value is 3
\end{DoxyItemize}\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md384}{}\doxysection{\texorpdfstring{EXIT STATUS}{EXIT STATUS}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md384}
{\ttfamily xxhsum} exit {\ttfamily 0} on success, {\ttfamily 1} if at least one file couldn\textquotesingle{}t be read or doesn\textquotesingle{}t have the same checksum as the {\ttfamily -\/c} option.\hypertarget{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md385}{}\doxysection{\texorpdfstring{EXAMPLES}{EXAMPLES}}\label{md_runtime_2xx_hash_2cli_2xxhsum_81_autotoc_md385}
Output xx\+Hash (64bit) checksum values of specific files to standard output \begin{DoxyVerb}$ xxhsum -H1 foo bar baz
\end{DoxyVerb}


Output xx\+Hash (32bit and 64bit) checksum values of specific files to standard output, and redirect it to {\ttfamily xyz.\+xxh32} and {\ttfamily qux.\+xxh64} \begin{DoxyVerb}$ xxhsum -H0 foo bar baz > xyz.xxh32
$ xxhsum -H1 foo bar baz > qux.xxh64
\end{DoxyVerb}


Read xx\+Hash sums from specific files and check them \begin{DoxyVerb}$ xxhsum -c xyz.xxh32 qux.xxh64
\end{DoxyVerb}


Produce a list of files, then generate hashes for that list \begin{DoxyVerb}$ find . -type f -name '*.[ch]' > c-files.txt
$ xxhsum --files-from c-files.txt
\end{DoxyVerb}


Read the list of files from standard input to avoid the need for an intermediate file \begin{DoxyVerb}$ find . -type f -name '*.[ch]' | xxhsum --files-from -
\end{DoxyVerb}


Note that if shell expansion, length of argument list, clarity of use of spaces in filenames, etc allow it then the same output as the previous example can be generated like this \begin{DoxyVerb}$ xxhsum `find . -name '*.[ch]'`
\end{DoxyVerb}


Benchmark xx\+Hash algorithm. By default, {\ttfamily xxhsum} benchmarks xx\+Hash main variants on a synthetic sample of 100 KB, and print results into standard output. The first column is the algorithm, the second column is the source data size in bytes, the third column is the number of hashes generated per second (throughput), and finally the last column translates speed in megabytes per second. \begin{DoxyVerb}$ xxhsum -b
\end{DoxyVerb}


In the following example, the sample to hash is set to 16384 bytes, the variants to be benched are selected by their IDs, and each benchmark test is repeated 10 times, for increased accuracy. \begin{DoxyVerb}$ xxhsum -b1,2,3 -i10 -B16384
\end{DoxyVerb}
