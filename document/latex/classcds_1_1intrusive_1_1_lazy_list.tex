class cds\+::intrusive\+::\+Lazy\+List$<$ GC, T, Traits $>$}
Lazy ordered single-\/linked list 

\label{classcds_1_1intrusive_1_1_lazy_list_cds_intrusive_LazyList_hp}%
\Hypertarget{classcds_1_1intrusive_1_1_lazy_list_cds_intrusive_LazyList_hp}%
 Usually, ordered single-\/linked list is used as a building block for the hash table implementation. The complexity of searching is {\ttfamily O(\+N)}.

Source\+:
\begin{DoxyItemize}
\item \mbox{[}2005\mbox{]} Steve Heller, Maurice Herlihy, Victor Luchangco, Mark Moir, William N. Scherer III, and Nir Shavit "{}\+A Lazy Concurrent List-\/\+Based Set Algorithm"{}
\end{DoxyItemize}

The lazy list is based on an optimistic locking scheme for inserts and removes, eliminating the need to use the equivalent of an atomically markable reference. It also has a novel wait-\/free membership {\ttfamily find} operation that does not need to perform cleanup operations and is more efficient.

Template arguments\+:
\begin{DoxyItemize}
\item {\ttfamily GC} -\/ Garbage collector used. Note the {\ttfamily GC} must be the same as the GC used for item type {\ttfamily T} (see \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1node}{lazy\+\_\+list\+::node}).
\item {\ttfamily T} -\/ type to be stored in the list. The type must be based on \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1node}{lazy\+\_\+list\+::node} (for \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1base__hook}{lazy\+\_\+list\+::base\+\_\+hook}) or it must have a member of type \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1node}{lazy\+\_\+list\+::node} (for \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1member__hook}{lazy\+\_\+list\+::member\+\_\+hook}).
\item {\ttfamily Traits} -\/ type traits. See \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1traits}{lazy\+\_\+list\+::traits} for explanation. It is possible to declare option-\/based list with \doxylink{structcds_1_1intrusive_1_1lazy__list_1_1make__traits}{cds\+::intrusive\+::lazy\+\_\+list\+::make\+\_\+traits} metafunction instead of {\ttfamily Traits} template argument. Forward iterator 

The forward iterator for lazy list has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item to protect the value, the iterator contains a GC-\/specific guard + another guard is required locally for increment operator. For some GC ({\ttfamily \doxylink{namespacecds_1_1gc_af0523881f089593a7559d194ee2b5dd9}{gc\+::\+HP}}), a guard is limited resource per thread, so an exception (or assertion) "{}no free guard"{} may be thrown if a limit of guard count per thread is exceeded.
\item The iterator cannot be moved across thread boundary since it contains GC\textquotesingle{}s guard that is thread-\/private GC data.
\item Iterator ensures thread-\/safety even if you delete the item that iterator points to. However, in case of concurrent deleting operations it is no guarantee that you iterate all item in the list. Moreover, a crash is possible when you try to iterate the next element that has been deleted by concurrent thread.
\end{DoxyItemize}

\begin{DoxyWarning}{警告}
Use this iterator on the concurrent container for debugging purpose only. 
\end{DoxyWarning}

Unlinks the item {\ttfamily val} from the list 

The function searches the item {\ttfamily val} in the list and unlink it from the list if it is found and it is equal to {\ttfamily val}.

Difference between \doxylink{classcds_1_1intrusive_1_1_lazy_list_a7e5ad059410155179ce85be20085ddc3}{erase} and {\ttfamily unlink} functions\+: {\ttfamily erase} finds {\itshape a key} and deletes the item found. {\ttfamily unlink} finds an item by key and deletes it only if {\ttfamily val} is an item of that list, i.\+e. the pointer to item found is equal to {\ttfamily  \&val }.

The function returns {\ttfamily true} if success and {\ttfamily false} otherwise.
