\doxysection{Release 3.9.0 (2025/01/02)}
\hypertarget{release-3-9-0}{}\label{release-3-9-0}\index{Release 3.9.0 (2025/01/02)@{Release 3.9.0 (2025/01/02)}}
\hypertarget{release-3-9-0_release-3-9-0_summary}{}\doxysubsection{\texorpdfstring{Release Summary}{Release Summary}}\label{release-3-9-0_release-3-9-0_summary}
This release improves scheduling performance with a decentralized work-\/stealing strategy and enhances exception handling across all task types.\hypertarget{release-3-9-0_release-3-9-0_download}{}\doxysubsection{\texorpdfstring{Download}{Download}}\label{release-3-9-0_release-3-9-0_download}
To download the newest version of Taskflow, please clone the master branch from \href{https://github.com/taskflow/taskflow}{\texttt{ Taskflow\textquotesingle{}s Git\+Hub}}.\hypertarget{release-3-9-0_release-3-9-0_system_requirements}{}\doxysubsection{\texorpdfstring{System Requirements}{System Requirements}}\label{release-3-9-0_release-3-9-0_system_requirements}
To use Taskflow v3.\+9.\+0, you need a compiler that supports C++17\+:

\begin{DoxyItemize}
\item GNU C++ Compiler at least v8.\+4 with -\/std=c++17 \item Clang C++ Compiler at least v6.\+0 with -\/std=c++17 \item Microsoft Visual Studio at least v19.\+27 with /std\+:c++17 \item Apple\+Clang Xcode Version at least v12.\+0 with -\/std=c++17 \item Nvidia CUDA Toolkit and Compiler (nvcc) at least v11.\+1 with -\/std=c++17 \item Intel C++ Compiler at least v19.\+0.\+1 with -\/std=c++17 \item Intel DPC++ Clang Compiler at least v13.\+0.\+0 with -\/std=c++17\end{DoxyItemize}
Taskflow works on Linux, Windows, and Mac OS X.

\begin{DoxyAttention}{注意}
Although Taskflow supports primarily C++17, you can enable C++20 compilation through {\ttfamily -\/std=c++20} to achieve better performance due to new C++20 features.
\end{DoxyAttention}
\hypertarget{release-3-9-0_release-3-9-0_new_features}{}\doxysubsection{\texorpdfstring{New Features}{New Features}}\label{release-3-9-0_release-3-9-0_new_features}
\hypertarget{release-3-9-0_release-3-9-0_taskflow_core}{}\doxysubsubsection{\texorpdfstring{Taskflow Core}{Taskflow Core}}\label{release-3-9-0_release-3-9-0_taskflow_core}

\begin{DoxyItemize}
\item improved the core scheduling algorithm using a decentralized work-\/stealing strategy
\begin{DoxyItemize}
\item \doxylink{classtf_1_1_bounded_task_queue}{tf\+::\+Bounded\+Task\+Queue} to optimize per-\/thread work-\/stealing latency
\item \doxylink{classtf_1_1_unbounded_task_queue}{tf\+::\+Unbounded\+Task\+Queue} to handle overflowed tasks
\end{DoxyItemize}
\item enhanced \doxylink{classtf_1_1_runtime}{tf\+::\+Runtime} to support preemptible execution flows
\item optimized task storage by storing detached tasks in their original subflows
\item optimized the query efficiency for strong dependencies by embedding their values in node states
\item updated \doxylink{classtf_1_1_graph}{tf\+::\+Graph} to derive from a vector of unique pointers to nodes
\begin{DoxyItemize}
\item Graph node lifetimes are managed by std\+::unique\+\_\+ptr
\item Asynchronous task node lifetimes are managed by \doxylink{classtf_1_1_executor}{tf\+::\+Executor}.
\end{DoxyItemize}
\item expanded unit tests to include more exception handling scenarios
\item decoupled \doxylink{classtf_1_1_runtime}{tf\+::\+Runtime} from static task to accommodate distinct execution logic
\item removed the blocking behavior to avoid underutilized threads for the following tasks\+:
\begin{DoxyItemize}
\item module task (\href{https://github.com/taskflow/taskflow/issues/649}{\texttt{ \#649}})
\item subflow task
\item all parallel algorithms (through preemptible async tasks)
\end{DoxyItemize}
\item removed std\+::bind from asynchronous tasks to ensure proper constexpr switch
\item added compile-\/time macros to enable specific features
\begin{DoxyItemize}
\item {\ttfamily TF\+\_\+\+ENABLE\+\_\+\+TASK\+\_\+\+POOL} to enable the use of task pool
\end{DoxyItemize}
\item added taskflow execution through asynchronous tasking with \doxylink{namespacetf_a767288f0f103fec9a849237865198627}{tf\+::make\+\_\+module\+\_\+task}
\begin{DoxyItemize}
\item details can be referred to \doxylink{ModuleAlgorithm}{Module Algorithm}
\end{DoxyItemize}
\item added \doxylink{classtf_1_1_worker_interface}{tf\+::\+Worker\+Interface} for users to configure the behaviors of workers
\begin{DoxyItemize}
\item details can be referred to \doxylink{ExecuteTaskflow}{Executor}
\end{DoxyItemize}
\item added worker interface example and unit tests
\end{DoxyItemize}\hypertarget{release-3-9-0_release-3-9-0_utilities}{}\doxysubsubsection{\texorpdfstring{Utilities}{Utilities}}\label{release-3-9-0_release-3-9-0_utilities}
