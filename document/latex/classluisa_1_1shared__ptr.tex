\subsubsection*{template$<$typename T$>$\newline
class luisa\+::shared\+\_\+ptr$<$ T $>$}
\doxylink{classluisa_1_1shared__ptr}{shared\+\_\+ptr}

This class implements the C++11 \doxylink{classluisa_1_1shared__ptr}{shared\+\_\+ptr} template. A \doxylink{classluisa_1_1shared__ptr}{shared\+\_\+ptr} is like the C++ Standard Library \doxylink{classluisa_1_1unique__ptr}{unique\+\_\+ptr} except that it allows sharing of pointers between instances via reference counting. \doxylink{classluisa_1_1shared__ptr}{shared\+\_\+ptr} objects can safely be copied and can safely be used in C++ Standard Library containers such as std\+::vector or std\+::list.

This class is not thread safe in that you cannot use an instance of it from two threads at the same time and cannot use two separate instances of it, which own the same pointer, at the same time. Use standard multithread mutex techniques to address the former problems and use shared\+\_\+ptr\+\_\+mt to address the latter. Note that this is contrary to the C++11 standard.

As of this writing, arrays aren\textquotesingle{}t supported, but they are planned in the future based on the C++17 proposal\+: \href{http://isocpp.org/files/papers/N3920.html}{\texttt{ http\+://isocpp.\+org/files/papers/\+N3920.\+html}}
