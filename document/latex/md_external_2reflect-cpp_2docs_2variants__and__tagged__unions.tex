Sometimes you know that the JSON object can be one of several alternatives. For example, you might have several shapes like {\ttfamily Circle}, {\ttfamily Rectangle} or {\ttfamily Square}. For these kind of cases, the C++ standard library contains {\ttfamily std\+::variant} and it is supported by reflect-\/cpp.

However, we recommend you use {\ttfamily \doxylink{classrfl_1_1_variant}{rfl\+::\+Variant}} instead.

{\ttfamily \doxylink{classrfl_1_1_variant}{rfl\+::\+Variant}} behaves just like {\ttfamily std\+::variant}, but it compiles considerably faster, particularly for variants with many alternatives.

You can use the functions {\ttfamily \doxylink{namespacerfl_ac379420409251e5a3c0bea4a701daf0a}{rfl\+::get}}, {\ttfamily \doxylink{namespacerfl_ae888abfe9b03cb12abccf2205daae330}{rfl\+::get\+\_\+if}}, {\ttfamily \doxylink{namespacerfl_a1ed102b316943bd9b92c1aae0a9c2119}{rfl\+::holds\+\_\+alternative}}, {\ttfamily \doxylink{namespacerfl_ab818223fa1c483a720a5004db13c118b}{rfl\+::visit}}, {\ttfamily \doxylink{namespacerfl_a73ec33297a48daba571bebf8307393b2}{rfl\+::variant\+\_\+alternative\+\_\+t}} and {\ttfamily \doxylink{namespacerfl_a86e7b6cea946dc3d8da6f593afde59db}{rfl\+::variant\+\_\+size\+\_\+v}} to access the variant and they work the same way as their equivalents in the standard library.

This code will compile just fine and work as intended. However, there are several problems with this\+:

1) It is in inefficient\+: The parser has to read the fields for all of the different alternatives until it can\textquotesingle{}t find a required field in the JSON object. It will then move on to the next alternative. 2) It leads to confusing error messages\+: If none of the alternatives can be matched, you will get an error message telling you why each of the alternatives couldn\textquotesingle{}t be matched. Such error messages are very long-\/winding and hard to read. 3) It is dangerous. Imagine we had written {\ttfamily \doxylink{classrfl_1_1_variant}{rfl\+::\+Variant}\texorpdfstring{$<$}{<}Circle, Square, Rectangle\texorpdfstring{$>$}{>}} instead of {\ttfamily \doxylink{classrfl_1_1_variant}{rfl\+::\+Variant}\texorpdfstring{$<$}{<}Circle, Rectangle, Square\texorpdfstring{$>$}{>}}. This would mean that {\ttfamily Rectangle} could never be matched, because the fields in {\ttfamily Square} are a subset of {\ttfamily Rectangle}. This leads to very confusing bugs.\hypertarget{md_external_2reflect-cpp_2docs_2variants__and__tagged__unions_autotoc_md707}{}\doxysection{\texorpdfstring{Automatic tags}{Automatic tags}}\label{md_external_2reflect-cpp_2docs_2variants__and__tagged__unions_autotoc_md707}
