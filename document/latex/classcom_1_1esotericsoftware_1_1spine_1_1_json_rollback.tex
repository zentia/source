\doxysubsection{详细描述}
Takes \doxylink{namespace_spine}{Spine} JSON data and transforms it to work with an older version of \doxylink{namespace_spine}{Spine}. Target versions\+:~\newline
 2.\+1\+: supports going from version 3.\+3.\+xx to 2.\+1.\+27.~\newline
 3.\+7\+: supports going from version 3.\+8.\+xx to 3.\+7.\+94.~\newline
 3.\+8\+: supports going from version 4.\+0.\+xx to 3.\+8.\+99 (all curves become linear, separate timelines are lost, constraint translate Y and shear Y are lost). 

\doxylink{class_data}{Data} can be exported from a \doxylink{namespace_spine}{Spine} project, processed with \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_json_rollback}{Json\+Rollback}, then imported into an older version of \doxylink{namespace_spine}{Spine}. However, \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_json_rollback}{Json\+Rollback} may remove data for features not supported by the older \doxylink{namespace_spine}{Spine} version. Because of this, \doxylink{classcom_1_1esotericsoftware_1_1spine_1_1_json_rollback}{Json\+Rollback} is only intended for situations were work was accidentally done with a newer \doxylink{namespace_spine}{Spine} version and now needs to be imported into an older \doxylink{namespace_spine}{Spine} version (eg, if runtime support for the new version is not yet available). 

Animators should freeze their \doxylink{namespace_spine}{Spine} editor version to match the \doxylink{namespace_spine}{Spine} version supported by the runtime being used. Only when the runtime is updated to support a newer \doxylink{namespace_spine}{Spine} version should animators update their \doxylink{namespace_spine}{Spine} editor version to match.
