Michael\textquotesingle{}s hash set 

\label{classcds_1_1container_1_1_michael_hash_set_cds_nonintrusive_MichaelHashSet_hp}%
\Hypertarget{classcds_1_1container_1_1_michael_hash_set_cds_nonintrusive_MichaelHashSet_hp}%
 Source\+:
\begin{DoxyItemize}
\item \mbox{[}2002\mbox{]} Maged Michael "{}\+High performance dynamic lock-\/free hash tables and list-\/based sets"{}
\end{DoxyItemize}

Michael\textquotesingle{}s hash table algorithm is based on lock-\/free ordered list and it is very simple. The main structure is an array {\ttfamily T} of size {\ttfamily M}. Each element in {\ttfamily T} is basically a pointer to a hash bucket, implemented as a singly linked list. The array of buckets cannot be dynamically expanded. However, each bucket may contain unbounded number of items.

Template parameters are\+:
\begin{DoxyItemize}
\item {\ttfamily GC} -\/ Garbage collector used. You may use any \doxylink{group__cds__garbage__collector}{Garbage collector} from the {\ttfamily libcds} library. Note the {\ttfamily GC} must be the same as the {\ttfamily GC} used for {\ttfamily Ordered\+List} 
\item {\ttfamily Ordered\+List} -\/ ordered list implementation used as bucket for hash set, possible implementations\+: {\ttfamily \doxylink{classcds_1_1container_1_1_michael_list}{Michael\+List}}, {\ttfamily \doxylink{classcds_1_1container_1_1_lazy_list}{Lazy\+List}}, {\ttfamily \doxylink{classcds_1_1container_1_1_iterable_list}{Iterable\+List}}. The ordered list implementation specifies the type {\ttfamily T} to be stored in the hash-\/set, the comparing functor for the type {\ttfamily T} and other features specific for the ordered list.
\item {\ttfamily Traits} -\/ set traits, default is {\ttfamily \doxylink{namespacecds_1_1container_1_1michael__set_a3f0d1a7a128d9b5c54b0fb5d05179948}{michael\+\_\+set\+::traits}}. Forward iterator 

The forward iterator for Michael\textquotesingle{}s set has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item to protect the value, the iterator contains a GC-\/specific guard + another guard is required locally for increment operator. For some GC (like as {\ttfamily \doxylink{namespacecds_1_1gc_af0523881f089593a7559d194ee2b5dd9}{gc\+::\+HP}}), a guard is a limited resource per thread, so an exception (or assertion) "{}no free guard"{} may be thrown if the limit of guard count per thread is exceeded.
\item The iterator cannot be moved across thread boundary because it contains thread-\/private GC\textquotesingle{}s guard.
\end{DoxyItemize}

Iterator thread safety depends on type of {\ttfamily Ordered\+List\+:} 
\begin{DoxyItemize}
\item for {\ttfamily \doxylink{classcds_1_1container_1_1_michael_list}{Michael\+List}} and {\ttfamily \doxylink{classcds_1_1container_1_1_lazy_list}{Lazy\+List}\+:} iterator guarantees safety even if you delete the item that iterator points to because that item is guarded by hazard pointer. However, in case of concurrent deleting operations it is no guarantee that you iterate all item in the set. Moreover, a crash is possible when you try to iterate the next element that has been deleted by concurrent thread. Use this iterator on the concurrent container for debugging purpose only.
\item for {\ttfamily \doxylink{classcds_1_1container_1_1_iterable_list}{Iterable\+List}\+:} iterator is thread-\/safe. You may use it freely in concurrent environment.
\end{DoxyItemize}

Initialize hash set 

The Michael\textquotesingle{}s hash set is non-\/expandable container. You should point the average count of items {\ttfamily n\+Max\+Item\+Count} when you create an object. {\ttfamily n\+Load\+Factor} parameter defines average count of items per bucket and it should be small number between 1 and 10. Remember, since the bucket implementation is an ordered list, searching in the bucket is linear \mbox{[}{\ttfamily O(n\+Load\+Factor)}\mbox{]}.

The ctor defines hash table size as rounding {\ttfamily n\+Max\+Item\+Count / n\+Load\+Factor} up to nearest power of two.
