\doxysubsection{详细描述}
class to create a taskflow object 

\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} taskflow manages a task dependency graph where each task represents a callable object (e.\+g., @std\+\_\+lambda, @std\+\_\+function) and an edge represents a dependency between two tasks. \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} task is one of the following types\+:


\begin{DoxyEnumerate}
\item static task \+: the callable constructible from {\ttfamily std\+::function$<$void()$>$} 
\item subflow task \+: the callable constructible from {\ttfamily std\+::function$<$void(tf\+::\+Subflow\&)$>$} 
\item condition task \+: the callable constructible from {\ttfamily std\+::function$<$int()$>$} 
\item multi-\/condition task\+: the callable constructible from {\ttfamily std\+::function\texorpdfstring{$<$}{<}tf\+::\+Small\+Vector$<$int$>$()\texorpdfstring{$>$}{>}} 
\item module task \+: the task constructed from \doxylink{classtf_1_1_flow_builder_ac6f22228d4c2ea2e643c4b0d42c0e92a}{tf\+::\+Taskflow\+::composed\+\_\+of} {\ttfamily std\+::function$<$void(tf\+::\+Runtime\&)$>$} 
\end{DoxyEnumerate}

Each task is a basic computation unit and is run by one worker thread from an executor. The following example creates a simple taskflow graph of four static tasks, {\ttfamily \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A}}, {\ttfamily \doxylink{bench__gemm_8cpp_a37a83060ac796961b44991c836f083f7}{B}}, {\ttfamily C}, and {\ttfamily D}, where {\ttfamily \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A}} runs before {\ttfamily \doxylink{bench__gemm_8cpp_a37a83060ac796961b44991c836f083f7}{B}} and {\ttfamily C} and {\ttfamily D} runs after {\ttfamily \doxylink{bench__gemm_8cpp_a37a83060ac796961b44991c836f083f7}{B}} and {\ttfamily C}.

The taskflow object itself is NOT thread-\/safe. You should not modifying the graph while it is running, such as adding new tasks, adding new dependencies, and moving the taskflow to another.
