It is possible to skip the serialization and deserialization of fields using {\ttfamily \doxylink{namespacerfl_aa4884b177ba91a619110deafbaa9d900}{rfl\+::\+Skip}}.

Note that {\ttfamily \doxylink{namespacerfl_aa4884b177ba91a619110deafbaa9d900}{rfl\+::\+Skip}} is unsupported by all schemaful formats, like Avro.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct\ }Person\ \{}
\DoxyCodeLine{\ \ rfl::Rename<\textcolor{stringliteral}{"{}firstName"{}},\ std::string>\ first\_name;}
\DoxyCodeLine{\ \ rfl::Rename<\textcolor{stringliteral}{"{}lastName"{}},\ std::string>\ last\_name;}
\DoxyCodeLine{\ \ \mbox{\hyperlink{namespacerfl_aa4884b177ba91a619110deafbaa9d900}{rfl::Skip<std::string>}}\ town;}
\DoxyCodeLine{\ \ \mbox{\hyperlink{namespacetest__add__struct__name_adf083b09022b62d12ed8b8453317e043}{Age}}\ \mbox{\hyperlink{namespacectre_af1ab492eac22f0bfaebe58d37347ce28a7d637d275668ed6d41a9b97e6ad3a556}{age}};}
\DoxyCodeLine{\};}

\end{DoxyCode}


This means that the field {\ttfamily town} will be ignored\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{"{}firstName"{}:"{}Homer"{},"{}lastName"{}:"{}Simpson"{},"{}age"{}:45\}}

\end{DoxyCode}


If you only want to skip the deserialization, but not the serialization, you can use {\ttfamily \doxylink{namespacerfl_a3289120500dca5a405ec966a2f931076}{rfl\+::\+Skip\+Deserialization}}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct\ }Person\ \{}
\DoxyCodeLine{\ \ rfl::Rename<\textcolor{stringliteral}{"{}firstName"{}},\ std::string>\ first\_name;}
\DoxyCodeLine{\ \ rfl::Rename<\textcolor{stringliteral}{"{}lastName"{}},\ std::string>\ last\_name;}
\DoxyCodeLine{\ \ \mbox{\hyperlink{namespacerfl_a3289120500dca5a405ec966a2f931076}{rfl::SkipDeserialization<std::string>}}\ town;}
\DoxyCodeLine{\ \ \mbox{\hyperlink{namespacetest__add__struct__name_adf083b09022b62d12ed8b8453317e043}{Age}}\ \mbox{\hyperlink{namespacectre_af1ab492eac22f0bfaebe58d37347ce28a7d637d275668ed6d41a9b97e6ad3a556}{age}};}
\DoxyCodeLine{\};}

\end{DoxyCode}


If you only want to skip the serialization, but not the deserialization, you can use {\ttfamily \doxylink{namespacerfl_aeb9265971459cc34326d2c46a961d9a7}{rfl\+::\+Skip\+Serialization}}. Note that this implies that the serializied data cannot be deserialized, because the field {\ttfamily town} is expected during deserialization, but not serialized.
