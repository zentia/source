Vcpkg é um esforço da comunidade para construir um ecossistema produtivo e robusto de bibliotecas nativas -\/ suas contribuições são inestimáveis!\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md747}{}\doxysection{\texorpdfstring{Problemas (issues)}{Problemas (issues)}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md747}
\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} maneira mais fácil de contribuir é relatando problemas com {\ttfamily vcpkg.\+exe} ou um pacote existente no \href{https://github.com/Microsoft/vcpkg}{\texttt{ Git\+Hub}}. Ao relatar um problema com {\ttfamily vcpkg.\+exe}, certifique-\/se de indicar claramente\+:


\begin{DoxyItemize}
\item \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} configuração da máquina\+: "{}\+Estou usando a Atualização de Aniversário do Windows 10. Minha máquina está usando a localidade fr-\/fr. Executei com sucesso o \textquotesingle{}instal boost\textquotesingle{}."{}
\item As etapas para reproduzir\+: "{}\+I run \textquotesingle{}vcpkg list\textquotesingle{}"{}
\item O resultado esperado\+: "{}\+Eu esperava ver \textquotesingle{}boost\+:x86-\/windows\textquotesingle{}"{}
\item O resultado real\+: "{}\+Não recebo nenhuma saída"{} ou "{}\+Recebo uma caixa de diálogo de travamento"{}
\end{DoxyItemize}

Ao relatar um problema com um pacote, certifique-\/se de indicar claramente\+:


\begin{DoxyItemize}
\item \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} configuração da máquina (como acima)
\item Qual pacote e versão você está construindo\+: "{}opencv 3.\+1.\+0"{}
\item Quaisquer logs de erro relevantes do processo de compilação.
\end{DoxyItemize}\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md748}{}\doxysection{\texorpdfstring{Pull Requests}{Pull Requests}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md748}
Estamos felizes em aceitar solicitações de correções, recursos, novos pacotes e atualizações para pacotes existentes. Para evitar desperdício de tempo, recomendamos abrir um tópico para discutir se o PR que você está pensando em fazer será aceitável. Isso é duplamente verdadeiro para recursos e novos pacotes.\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md749}{}\doxysubsection{\texorpdfstring{Diretrizes de novos pacotes}{Diretrizes de novos pacotes}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md749}
Estamos felizes por você estar interessado em enviar um novo pacote! Aqui estão algumas diretrizes para ajudá-\/lo a criar um excelente portfile\+:


\begin{DoxyItemize}
\item Evite patches funcionais. Os patches devem ser considerados um último recurso para implementar a compatibilidade quando não houver outra maneira.
\item Quando os patches não puderem ser evitados, não modifique o comportamento padrão. O ciclo de vida ideal de um patch é ser mesclado no upstream e não ser mais necessário. Tente manter esse objetivo em mente ao decidir como corrigir algo.
\item Prefira usar as funções {\ttfamily vcpkg\+\_\+xyz} em vez de chamadas brutas {\ttfamily execute\+\_\+command}. Isso facilita a manutenção de longo prazo quando novos recursos (como sinalizadores de compilador personalizados ou geradores) são adicionados.
\end{DoxyItemize}\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md750}{}\doxysection{\texorpdfstring{Jurídico}{Jurídico}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__pt_autotoc_md750}
Você precisará preencher um Contrato de Licença de Colaborador (CLA) antes que sua solicitação pull possa ser aceita. Este contrato atesta que você está nos concedendo permissão para usar o código-\/fonte que está enviando e que este trabalho está sendo enviado sob a licença apropriada para que possamos usá-\/lo.

Você pode concluir o CLA seguindo as etapas em \href{https://cla.microsoft.com}{\texttt{ https\+://cla.\+microsoft.\+com}}. Assim que recebermos o CLA assinado, analisaremos a solicitação. Você só precisará fazer isso uma vez.
