\doxysection{Task-\/parallel Scalable Pipeline}
\hypertarget{_task_parallel_scalable_pipeline}{}\label{_task_parallel_scalable_pipeline}\index{Task-\/parallel Scalable Pipeline@{Task-\/parallel Scalable Pipeline}}
Unlike \doxylink{classtf_1_1_pipeline}{tf\+::\+Pipeline} (see \doxylink{TaskParallelPipeline}{Task-\/parallel Pipeline}) that instantiates all pipes at the construction time, Taskflow provides a scalable alternative called \doxylink{classtf_1_1_scalable_pipeline}{tf\+::\+Scalable\+Pipeline} to allow variable assignments of pipes using range iterators. \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} scalable pipeline is thus more flexible for applications to create a pipeline scheduling framework whose pipeline structure depends on runtime variables.\hypertarget{_task_parallel_scalable_pipeline_IncludeTheScalablePipelineHeader}{}\doxysubsection{\texorpdfstring{Include the Header}{Include the Header}}\label{_task_parallel_scalable_pipeline_IncludeTheScalablePipelineHeader}
You need to include the header file, {\ttfamily taskflow/algorithm/pipeline.hpp}, for creating a scalable pipeline scheduling framework.

\hypertarget{_task_parallel_scalable_pipeline_CreateAScalablePipelineModuleTask}{}\doxysubsection{\texorpdfstring{Create a Scalable Pipeline Module Task}{Create a Scalable Pipeline Module Task}}\label{_task_parallel_scalable_pipeline_CreateAScalablePipelineModuleTask}
Similar to \doxylink{classtf_1_1_pipeline}{tf\+::\+Pipeline}, \doxylink{classtf_1_1_scalable_pipeline}{tf\+::\+Scalable\+Pipeline} is a composable graph object to implement a {\itshape pipeline scheduling framework} in a taskflow. The key difference between \doxylink{classtf_1_1_pipeline}{tf\+::\+Pipeline} and \doxylink{classtf_1_1_scalable_pipeline}{tf\+::\+Scalable\+Pipeline} is that a scalable pipeline can accept {\itshape variable} assignments of pipes rather than instantiating all pipes at construction or programming time. Users define a linear range of pipes, each of the same callable type, and apply that range to construct a scalable pipeline. Between successive runs, users can reset the pipeline to a different range of pipes. \begin{DoxyAttention}{注意}
Unlike \doxylink{classtf_1_1_pipeline}{tf\+::\+Pipeline} that keeps the given pipes in a std\+::tuple object, \doxylink{classtf_1_1_scalable_pipeline}{tf\+::\+Scalable\+Pipeline} does not own the given pipe but maintains a vector of iterators to each pipe in the given range. It is your responsibility to keep those pipe objects alive during the execution of the pipeline task.

\hypertarget{_task_parallel_scalable_pipeline_ScalablePipelineUseOtherIteratorTypes}{}\doxysubsection{\texorpdfstring{Use Other Iterator Types}{Use Other Iterator Types}}\label{_task_parallel_scalable_pipeline_ScalablePipelineUseOtherIteratorTypes}
When assigning a range to a scalable pipeline, the pipeline fetches all pipe iterators in that range to an internal vector. This organization allows invoking a pipe callable to be a random accessible operation, regardless of the pipe container type. Taskflow does not have much restriction on the iterator type, as long as these pipes can be iterated in a sequential order using the postfix increment operator, {\ttfamily ++}.
