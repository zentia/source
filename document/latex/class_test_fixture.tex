\subsubsection*{template$<$typename Concrete\+Test\+Fixture$>$\newline
class Test\+Fixture$<$ Concrete\+Test\+Fixture $>$}
\doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} \doxylink{class_test_fixture}{Test\+Fixture} is a class that contain \doxylink{class_test_fixture_a337c0f6479b1bc9c584f581a0c839e99}{Test\+Cases} --which corresponds to Concrete\+Test\+Fixture methods-- common objects uder tests, and set\+Up and tear\+Down methods which are automatically executed before and after each test case.

Is the base class of Concrete\+Fixtures implemented by the framework user

It does the \textquotesingle{}Composite\textquotesingle{} role in the \textquotesingle{}Composite\textquotesingle{} GoF pattern. Its composite children are \doxylink{class_test_fixture_a337c0f6479b1bc9c584f581a0c839e99}{Test\+Cases}, which wrapps the test methods.

It is a template class parametrized by Concrete\+Test\+Fixture so that it can instantiate \doxylink{class_test_fixture_1_1_test_case}{Test\+Case} objects templatized with this same parameter\+: it needs the concrete class type for calling its non-\/static methods.
