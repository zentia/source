\doxysection{Release 2.6.0 (2020/08/25)}
\hypertarget{release-2-6-0}{}\label{release-2-6-0}\index{Release 2.6.0 (2020/08/25)@{Release 2.6.0 (2020/08/25)}}
Taskflow 2.\+6.\+0 is the 8th release in the 2.\+x line! This release includes several new changes such as CPU-\/\+GPU tasking, algorithm collection, enhanced web-\/based profiler, documentation, and unit tests.

We have a new \href{https://taskflow.github.io/}{\texttt{ webpage}} for Taskflow!\hypertarget{release-2-6-0_release-2-6-0_download}{}\doxysubsection{\texorpdfstring{Download}{Download}}\label{release-2-6-0_release-2-6-0_download}
Taskflow 2.\+6.\+0 can be downloaded from \href{https://github.com/taskflow/taskflow/releases/tag/v2.6.0}{\texttt{ here}}.\hypertarget{release-2-6-0_release-2-6-0_new_features}{}\doxysubsection{\texorpdfstring{New Features}{New Features}}\label{release-2-6-0_release-2-6-0_new_features}
\begin{DoxyItemize}
\item added explicit join behavior of \doxylink{classtf_1_1_subflow}{tf\+::\+Subflow} (see \doxysectlink{_subflow_tasking_JoinASubflow}{Join a Subflow}{1} and \doxylink{fibonacci}{Fibonacci Number}) \item added version macro ({\ttfamily TF\+\_\+\+VERSION}, {\ttfamily TF\+\_\+\+MAJOR\+\_\+\+VERSION}, {\ttfamily TF\+\_\+\+MINOR\+\_\+\+VERSION}, {\ttfamily TF\+\_\+\+PATCH\+\_\+\+VERSION}) to retrieve version info programmatically (\doxylink{namespacetf_a06790e5f6898894392f247309626e1b4}{tf\+::version}) \item added {\ttfamily TF\+\_\+\+BUILD\+\_\+\+TESTS} and {\ttfamily TF\+\_\+\+BUILD\+\_\+\+EXAMPLES} (default on) to let users disable the build of tests and examples (see \doxysectlink{install}{Building and Installing}{0}) \item renamed tf\+::\+Taskflkow\+::parallel\+\_\+for to \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each} to follow the STL convention \item redesigned \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each} and \doxylink{classtf_1_1_flow_builder_a3b132bd902331a11b04b4ad66cf8bf77}{tf\+::\+Taskflow\+::for\+\_\+each\+\_\+index} using Open\+MP-\/styled scheduling algorithms; this redesign largely improved the performance of parallel-\/for using a single dynamic task return, but it breaks the previous API that returned a std\+::pair of tasks to synchronize on a set of static parallel-\/for tasks. Yet, we believe adopting this change is not difficult (see \doxylink{ParallelIterations}{Parallel Iterations}). \item added multiple unit tests for \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each} and \doxylink{classtf_1_1_flow_builder_a3b132bd902331a11b04b4ad66cf8bf77}{tf\+::\+Taskflow\+::for\+\_\+each\+\_\+index} at different partition algorithms; we have implemented our partition algorithms based on the Open\+MP library implementation of LLVM and GCC. \item added Mandelbrot application in the benchmark to evaluate the performance of parallel-\/for \item redesigned \doxylink{classtf_1_1_flow_builder_afb24798ebf46e253a40b01bffb1da6a7}{tf\+::\+Taskflow\+::reduce} and \doxylink{classtf_1_1_flow_builder_aa62d24438c0860e76153ffd129deba41}{tf\+::\+Taskflow\+::transform\+\_\+reduce} based on the parallel architecture of \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each} (see \doxylink{ParallelReduction}{Parallel Reduction}).\end{DoxyItemize}
