Extending spine-\/c requires implementing three methods\+:


\begin{DoxyItemize}
\item {\ttfamily \+\_\+sp\+Atlas\+Page\+\_\+create\+Texture} Loads a texture and stores it and its size in the {\ttfamily void\texorpdfstring{$\ast$}{*} renderer\+Object}, {\ttfamily width} and {\ttfamily height} fields of an {\ttfamily \doxylink{structsp_atlas_page}{sp\+Atlas\+Page}} struct.
\item {\ttfamily \+\_\+sp\+Atlas\+Page\+\_\+dispose\+Texture} Disposes of a texture loaded with {\ttfamily \+\_\+sp\+Atlas\+Page\+\_\+create\+Texture}.
\item {\ttfamily \+\_\+sp\+Util\+\_\+read\+File} Reads a file. If this doesn\textquotesingle{}t need to be customized, {\ttfamily \+\_\+read\+File} is provided which reads a file using {\ttfamily fopen}.
\end{DoxyItemize}

With these implemented, the spine-\/c API can then be used to load \doxylink{namespace_spine}{Spine} animation data. Rendering is done by enumerating the slots for a skeleton and rendering the attachment for each slot. Each attachment has a {\ttfamily renderer\+Object} field that is set when the attachment is loaded.

For example, {\ttfamily Atlas\+Attachment\+Loader} is typically used to load attachments when using a \doxylink{namespace_spine}{Spine} texture atlas. When {\ttfamily Atlas\+Attachment\+Loader} loads a {\ttfamily Region\+Attachment}, the attachment\textquotesingle{}s {\ttfamily void\texorpdfstring{$\ast$}{*} renderer\+Object} is set to an {\ttfamily Atlas\+Region}. Rendering code can then obtain the {\ttfamily Atlas\+Region} from the attachment, get the {\ttfamily Atlas\+Page} it belongs to, and get the page\textquotesingle{}s {\ttfamily void\texorpdfstring{$\ast$}{*} renderer\+Object}. This is the renderer specific texture object set by {\ttfamily \+\_\+sp\+Atlas\+Page\+\_\+create\+Texture}. Attachment loading can be \href{http://esotericsoftware.com/spine-using-runtimes/\#attachmentloader}{\texttt{ customized}} if not using {\ttfamily Atlas\+Attachment\+Loader} or to provider different renderer specific data.

\href{../spine-sfml/src/c/spine/spine-sfml.cpp\#L39}{\texttt{ spine-\/sfml}} serves as a simple example of extending spine-\/c.

spine-\/c uses an OOP style of programming where each "{}class"{} is made up of a struct and a number of functions prefixed with the struct name.
