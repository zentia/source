\subsubsection*{template$<$typename T, typename Allocator = EASTLAllocator\+Type, unsigned k\+Deque\+Subarray\+Size = DEQUE\+\_\+\+DEFAULT\+\_\+\+SUBARRAY\+\_\+\+SIZE(\+T)$>$\newline
class eastl\+::deque$<$ T, Allocator, k\+Deque\+Subarray\+Size $>$}
deque

Implements a conventional C++ double-\/ended queue. The implementation used here is very much like any other deque implementations you may have seen, as it follows the standard algorithm for deque design.

Note\+: As of this writing, deque does not support zero-\/allocation initial emptiness. A newly created deque with zero elements will still allocate a subarray pointer set. We are looking for efficient and clean ways to get around this, but current efforts have resulted in less efficient and more fragile code. The logic of this class doesn\textquotesingle{}t lend itself to a clean implementation. It turns out that deques are one of the least likely classes you\textquotesingle{}d want this behaviour in, so until this functionality becomes very important to somebody, we will leave it as-\/is. It can probably be solved by adding some extra code to the Do\texorpdfstring{$\ast$}{*} functions and adding good comments explaining the situation.
