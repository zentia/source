\doxysubsection{详细描述}
Elimination technique 

\label{namespacecds_1_1algo_1_1elimination_cds_elimination_description}%
\Hypertarget{namespacecds_1_1algo_1_1elimination_cds_elimination_description}%
Elimination technique allows highly distributed coupling and execution of operations with reverse semantics like the pushes and pops on a stack. If a push followed by a pop are performed on a stack, the data structure\textquotesingle{}s state does not change (similarly for a pop followed by a push). This means that if one can cause pairs of pushes and pops to meet and pair up in separate locations, the threads can exchange values without having to touch a centralized structure since they have anyhow "{}eliminated"{} each other\textquotesingle{}s effect on it. Elimination can be implemented by using a collision array in which threads pick random locations in order to try and collide. Pairs of threads that "{}collide"{} in some location run through a synchronization protocol, and all such disjoint collisions can be performed in parallel. If a thread has not met another in the selected location or if it met a thread with an operation that cannot be eliminated (such as two push operations), an alternative scheme must be used.

For each thread an struct {\ttfamily \doxylink{structcds_1_1algo_1_1elimination_1_1record}{cds\+::algo\+::elimination\+::record}} stores current elimination record. TLS-\/based {\ttfamily \doxylink{classcds_1_1algo_1_1elimination_1_1storage}{cds\+::algo\+::elimination\+::storage}} class manages per-\/thread elimination record.
