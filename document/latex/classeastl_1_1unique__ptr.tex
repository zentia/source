class \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr}

This class implements a \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr} template. This is a class which is similar to the C++ auto\+\_\+ptr template, except that it prohibits copying of itself, for safety.

More specifically, the \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr} class template stores a pointer to a dynamically allocated object. The object pointed to is automatically deleted on destructor of \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr} or can be manually deleted via the \doxylink{classeastl_1_1unique__ptr_a6f0700a32b43641960ea6be25a521331}{unique\+\_\+ptr\+::reset} function.

Memory allocation notes\+: \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr} doesn\textquotesingle{}t allocate memory; all allocated pointers are externally derived. \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr} does deallocate memory, though always through the user-\/provided deleter. You need to make sure you are consistent in providing a deleter which frees memory in a way that matches how it was originally allocated. Deleters have instance information and are moved between containers the same way the allocated pointers are. Thus you can allocate memory via some heap and provide a deleter which contains a pointer to that same heap, and regardless of what you do with the \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr}, including moving it to another \doxylink{classeastl_1_1unique__ptr}{unique\+\_\+ptr}, the deletion will use the originally provided heap.
