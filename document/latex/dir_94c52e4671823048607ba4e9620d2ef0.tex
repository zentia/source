\doxysubsection{详细描述}
This Git\+Hub project hosts the \doxylink{namespace_spine}{Spine} Runtimes which are needed to use \href{http://esotericsoftware.com/}{\texttt{ Spine}} 2D skeletal animation data with various game toolkits.\hypertarget{README.md_autotoc_md924}{}\doxysubsection{\texorpdfstring{Licensing}{Licensing}}\label{README.md_autotoc_md924}
You are welcome to evaluate the \doxylink{namespace_spine}{Spine} Runtimes and the examples we provide in this repository free of charge.

You can integrate the \doxylink{namespace_spine}{Spine} Runtimes into your software free of charge, but users of your software must have their own \href{https://esotericsoftware.com/spine-purchase}{\texttt{ Spine license}}. Please make your users aware of this requirement! This option is often chosen by those making development tools, such as an SDK, game toolkit, or software library.

In order to distribute your software containing the \doxylink{namespace_spine}{Spine} Runtimes to others that don\textquotesingle{}t have a \doxylink{namespace_spine}{Spine} license, you need a \href{https://esotericsoftware.com/spine-purchase}{\texttt{ Spine license}} at the time of integration. Then you can distribute your software containing the \doxylink{namespace_spine}{Spine} Runtimes however you like, provided others don\textquotesingle{}t modify it or use it to create new software. If others want to do that, they\textquotesingle{}ll need their own \doxylink{namespace_spine}{Spine} license.

For the official legal terms governing the \doxylink{namespace_spine}{Spine} Runtimes, please read the \href{http://esotericsoftware.com/spine-runtimes-license}{\texttt{ Spine Runtimes License Agreement}} and Section 2 of the \href{http://esotericsoftware.com/spine-editor-license\#s2}{\texttt{ Spine Editor License Agreement}}.\hypertarget{README.md_autotoc_md925}{}\doxysubsection{\texorpdfstring{Documentation}{Documentation}}\label{README.md_autotoc_md925}
See the \href{http://esotericsoftware.com/spine-runtimes-guide}{\texttt{ Spine Runtimes Guide}} for detailed information about using the \doxylink{namespace_spine}{Spine} Runtimes. The \doxylink{namespace_spine}{Spine} \href{http://esotericsoftware.com/spine-documentation\#runtimesTitle}{\texttt{ documentation page}} provides further information about tools and data formats. For runtime specific documentation, refer to the {\ttfamily README.\+md} file in each runtime directory.\hypertarget{README.md_autotoc_md926}{}\doxysubsection{\texorpdfstring{Bugs, enhancements, and tasks}{Bugs, enhancements, and tasks}}\label{README.md_autotoc_md926}
Review our backlog of bugs, enhancements, and tasks in the \href{https://github.com/EsotericSoftware/spine-runtimes/issues}{\texttt{ spine-\/runtimes}} and \href{https://github.com/EsotericSoftware/spine-editor/issues}{\texttt{ spine-\/editor}} issue trackers. Our \href{http://esotericsoftware.com/spine-roadmap}{\texttt{ roadmap}} provides a more convenient view of the same issue tracker information.\hypertarget{README.md_autotoc_md927}{}\doxysubsection{\texorpdfstring{Versioning}{Versioning}}\label{README.md_autotoc_md927}
The default branch on Git\+Hub is stable and works with data exported from the latest, non-\/beta version of the \doxylink{namespace_spine}{Spine} editor. New development is done in an {\ttfamily X.\+X-\/beta} branch, which may be a work in progress. Important changes to the runtimes can be reviewed in the \doxysectlink{md_external_2spine-runtimes_2_c_h_a_n_g_e_l_o_g}{CHANGELOG.md file}{0}. Occasionally the \doxylink{namespace_spine}{Spine} Runtimes are \href{https://github.com/EsotericSoftware/spine-runtimes/releases}{\texttt{ tagged}} with the specific \doxylink{namespace_spine}{Spine} editor version they work with.

It is highly suggested to \href{http://esotericsoftware.com/spine-settings\#Version}{\texttt{ freeze}} the \doxylink{namespace_spine}{Spine} editor version to match the \doxylink{namespace_spine}{Spine} Runtimes source being used and to update them in lock step. See the \href{http://esotericsoftware.com/spine-runtime-architecture\#Versioning}{\texttt{ Spine Runtimes Guide}} for more information.\hypertarget{README.md_autotoc_md928}{}\doxysubsection{\texorpdfstring{Contributing}{Contributing}}\label{README.md_autotoc_md928}
In order to merge your contributions, we need a signed \href{http://esotericsoftware.com/licenses/cla.txt}{\texttt{ contributor license agreement (CLA)}} from you. You can send a copy of the CLA to \href{mailto:contact@esotericsoftware.com}{\texttt{ contact@esotericsoftware.\+com}}.

When possible, it is best to base your contributions on the current beta branch ({\ttfamily X.\+X-\/beta}). Please be sure to follow the style and formatting you find in the respective runtime code to which you are contributing.
