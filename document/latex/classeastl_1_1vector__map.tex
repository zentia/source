\doxysubsection{详细描述}
\subsubsection*{template$<$typename Key, typename T, typename \mbox{\hyperlink{namespaceeastl_ae3d33fc51c4c15a85aee95335a5d336d}{Compare}} = eastl\+::less$<$\+Key$>$, typename Allocator = EASTLAllocator\+Type, typename Random\+Access\+Container = eastl\+::vector$<$eastl\+::pair$<$\+Key, T$>$, Allocator$>$$>$\newline
class eastl\+::vector\+\_\+map$<$ Key, T, Compare, Allocator, Random\+Access\+Container $>$}
\doxylink{classeastl_1_1vector__map}{vector\+\_\+map}

Implements a map via a random access container such as a vector.

Note that with \doxylink{classeastl_1_1vector__set}{vector\+\_\+set}, \doxylink{classeastl_1_1vector__multiset}{vector\+\_\+multiset}, \doxylink{classeastl_1_1vector__map}{vector\+\_\+map}, \doxylink{classeastl_1_1vector__multimap}{vector\+\_\+multimap} that the modification of the container potentially invalidates all existing iterators into the container, unlike what happens with conventional sets and maps.

This type could conceptually use a \doxylink{structeastl_1_1array}{eastl\+::array} as its underlying container, however the current design requires an allocator aware container. Consider using a \doxylink{classeastl_1_1fixed__vector}{fixed\+\_\+vector} instead.

Note that we set the \doxylink{classeastl_1_1vector__map_ad5116da2df39da172b95bd83c2023d88}{value\+\_\+type} to be \doxylink{structeastl_1_1pair}{pair$<$\+Key, T$>$} and not \doxylink{structeastl_1_1pair}{pair$<$const Key, T$>$}. This means that the underlying container (e.\+g vector) is a container of \doxylink{structeastl_1_1pair}{pair$<$\+Key, T$>$}. Our vector and deque implementations are optimized to assign values in-\/place and using a vector of \doxylink{structeastl_1_1pair}{pair$<$const Key, T$>$} (note the const) would make it hard to use our existing vector implementation without a lot of headaches. As a result, at least for the time being we do away with the const. This means that the insertion type varies between map and \doxylink{classeastl_1_1vector__map}{vector\+\_\+map} in that the latter doesn\textquotesingle{}t take const. This also means that a certain amount of automatic safety provided by the implementation is lost, as the compiler will let the wayward user modify a key and thus make the container no longer ordered behind its back.
