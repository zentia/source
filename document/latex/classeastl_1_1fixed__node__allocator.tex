\subsubsection*{template$<$size\+\_\+t node\+Size, size\+\_\+t node\+Count, size\+\_\+t node\+Alignment, size\+\_\+t node\+Alignment\+Offset, \mbox{\hyperlink{sugar_8h_abb452686968e48b67397da5f97445f5b}{bool}} b\+Enable\+Overflow, typename Overflow\+Allocator = EASTLAllocator\+Type$>$\newline
class eastl\+::fixed\+\_\+node\+\_\+allocator$<$ node\+Size, node\+Count, node\+Alignment, node\+Alignment\+Offset, b\+Enable\+Overflow, Overflow\+Allocator $>$}
\doxylink{classeastl_1_1fixed__node__allocator}{fixed\+\_\+node\+\_\+allocator}

Note\+: This class was previously named fixed\+\_\+node\+\_\+pool, but was changed because this name was inconsistent with the other allocators here which ended with \+\_\+allocator.

Implements a \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool} with a given node count, alignment, and alignment offset. \doxylink{classeastl_1_1fixed__node__allocator}{fixed\+\_\+node\+\_\+allocator} is like \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool} except it is templated on the node type instead of being a generic allocator. All it does is pass allocations through to the \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool} base. This functionality is separate from \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool} because there are other uses for \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool}.

We template on k\+Node\+Size instead of node\+\_\+type because the former allows for the two different node\+\_\+types of the same size to use the same template implementation.

Template parameters\+: node\+Size The size of the object to allocate. node\+Count The number of objects the pool contains. node\+Alignment The alignment of the objects to allocate. node\+Alignment\+Offset The alignment offset of the objects to allocate. b\+Enable\+Overflow Whether or not we should use the overflow heap if our object pool is exhausted. Overflow\+Allocator Overflow allocator, which is only used if b\+Enable\+Overflow == true. Defaults to the global heap.

\doxylink{classeastl_1_1fixed__node__allocator}{fixed\+\_\+node\+\_\+allocator}

Note that we are copying x.\+mp\+Head to our own \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool}. This at first may seem broken, as fixed pools cannot take over ownership of other fixed pools\textquotesingle{} memory. However, we declare that this copy ctor can only ever be safely called when the user has intentionally pre-\/seeded the source with the destination pointer. This is somewhat playing with fire, but it allows us to get around chicken-\/and-\/egg problems with containers being their own allocators, without incurring any memory costs or extra code costs. There\textquotesingle{}s another reason for this\+: we very strongly want to avoid full copying of instances of \doxylink{classeastl_1_1fixed__pool}{fixed\+\_\+pool} around, especially via the stack. Larger pools won\textquotesingle{}t even be able to fit on many machine\textquotesingle{}s stacks. So this solution is also a mechanism to prevent that situation from existing and being used. Perhaps some day we\textquotesingle{}ll find a more elegant yet costless way around this.
