\subsubsection*{template$<$typename Scalar\+\_\+, \mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}} Num\+Indices\+\_\+, \mbox{\hyperlink{sugar_8h_ae4f82344f573c70aa1066a2394bba345}{int}} Options\+\_\+, typename Index\+Type\+\_\+$>$\newline
class Tensor$<$ Scalar\+\_\+, Num\+Indices\+\_\+, Options\+\_\+, Index\+Type\+\_\+ $>$}
The tensor class. 

The Tensor class is the work-\/horse for all {\itshape dense} tensors within \doxylink{namespace_eigen}{Eigen}.

The Tensor class encompasses only dynamic-\/size objects so far.

The first two template parameters are required\+: 
\begin{DoxyTemplParams}{模板参数}
{\em Scalar\+\_\+} & Numeric type, e.\+g. float, double, int or {\ttfamily std\+::complex\texorpdfstring{$<$}{<}float\texorpdfstring{$>$}{>}}. User defined scalar types are supported as well (see \doxylink{_topic_customizing__custom_scalar_user_defined_scalars}{here}). \\
\hline
{\em Num\+Indices\+\_\+} & \doxylink{test__concurrent__vector__v2_8cpp_a36bc74cc1ee36b525959509e2a81c81e}{Number} of indices (i.\+e. rank of the tensor)\\
\hline
\end{DoxyTemplParams}
The remaining template parameters are optional -- in most cases you don\textquotesingle{}t have to worry about them. 
\begin{DoxyTemplParams}{模板参数}
{\em Options\+\_\+} & \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} combination of either {\bfseries{\doxylink{cxx11__tensor__assign_8cpp_a77c993a8d9f6efe5c1159fb2ab07dd4f}{Row\+Major}}} or {\bfseries{\#\+Col\+Major}}, and of either {\bfseries{\#\+Auto\+Align}} or {\bfseries{\#\+Dont\+Align}}. The former controls \doxylink{group___topic_storage_orders}{storage order}, and defaults to column-\/major. The latter controls alignment, which is required for vectorization. It defaults to aligning tensors. Note that tensors currently do not support any operations that profit from vectorization. Support for such operations (i.\+e. adding two tensors etc.) is planned.\\
\hline
\end{DoxyTemplParams}
You can access elements of tensors using normal subscripting\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Eigen::Tensor<double,\ 4>\ \mbox{\hyperlink{boing_8c_a87accd1af8e0aff4b818d891374f7cec}{t}}(10,\ 10,\ 10,\ 10);}
\DoxyCodeLine{\mbox{\hyperlink{boing_8c_a87accd1af8e0aff4b818d891374f7cec}{t}}(0,\ 1,\ 2,\ 3)\ =\ 42.0;}

\end{DoxyCode}


This class can be extended with the help of the plugin mechanism described on the page \doxylink{TopicCustomizing_Plugins}{Extending Matrix\+Base (and other classes)} by defining the preprocessor symbol {\ttfamily EIGEN\+\_\+\+TENSOR\+\_\+\+PLUGIN}.

{\itshape {\bfseries{Some notes\+:}}}


\begin{DoxyDescription}
\item[{\bfseries{Relation to other parts of \doxylink{namespace_eigen}{Eigen}\+:}} ]\hfill \\
The midterm development goal for this class is to have a similar hierarchy as \doxylink{namespace_eigen}{Eigen} uses for matrices, so that taking blocks or using tensors in expressions is easily possible, including an interface with the vector/matrix code by providing .as\+Matrix() and .as\+Vector() (or similar) methods for rank 2 and 1 tensors. However, currently, the Tensor class does not provide any of these features and is only available as a stand-\/alone class that just allows for coefficient access. Also, when fixed-\/size tensors are implemented, the number of template arguments is likely to change dramatically. 
\end{DoxyDescription}
