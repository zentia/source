class cds\+::container\+::\+Michael\+List$<$ GC, T, Traits $>$}
Michael\textquotesingle{}s ordered list 

\label{classcds_1_1container_1_1_michael_list_cds_nonintrusive_MichaelList_gc}%
\Hypertarget{classcds_1_1container_1_1_michael_list_cds_nonintrusive_MichaelList_gc}%
 Usually, ordered single-\/linked list is used as a building block for the hash table implementation. The complexity of searching is {\ttfamily O(\+N)}, where {\ttfamily N} is the item count in the list, not in the hash table.

Source\+:
\begin{DoxyItemize}
\item \mbox{[}2002\mbox{]} Maged Michael "{}\+High performance dynamic lock-\/free hash tables and list-\/based sets"{}
\end{DoxyItemize}

This class is non-\/intrusive version of \doxylink{classcds_1_1intrusive_1_1_michael_list}{cds\+::intrusive\+::\+Michael\+List} class

Template arguments\+:
\begin{DoxyItemize}
\item {\ttfamily GC} -\/ garbage collector used
\item {\ttfamily T} -\/ type stored in the list. The type must be default-\/ and copy-\/constructible.
\item {\ttfamily Traits} -\/ type traits, default is {\ttfamily \doxylink{structcds_1_1container_1_1michael__list_1_1traits}{michael\+\_\+list\+::traits}} 
\end{DoxyItemize}

Unlike standard container, this implementation does not divide type {\ttfamily T} into key and value part and may be used as a main building block for hash set algorithms. The key is a function (or a part) of type {\ttfamily T}, and this function is specified by {\ttfamily Traits\+::compare} functor or {\ttfamily Traits\+::less} predicate

\doxylink{classcds_1_1container_1_1_michael_k_v_list}{Michael\+KVList} is a key-\/value version of Michael\textquotesingle{}s non-\/intrusive list that is closer to the C++ std library approach.

Forward iterator 

The forward iterator for Michael\textquotesingle{}s list has some features\+:
\begin{DoxyItemize}
\item it has no post-\/increment operator
\item to protect the value, the iterator contains a GC-\/specific guard + another guard is required locally for increment operator. For some GC ({\ttfamily \doxylink{namespacecds_1_1gc_af0523881f089593a7559d194ee2b5dd9}{gc\+::\+HP}}), a guard is limited resource per thread, so an exception (or assertion) "{}no free guard"{} may be thrown if a limit of guard count per thread is exceeded.
\item The iterator cannot be moved across thread boundary since it contains GC\textquotesingle{}s guard that is thread-\/private GC data.
\item Iterator ensures thread-\/safety even if you delete the item that iterator points to. However, in case of concurrent deleting operations it is no guarantee that you iterate all item in the list. Moreover, a crash is possible when you try to iterate the next element that has been deleted by concurrent thread.
\end{DoxyItemize}

\begin{DoxyWarning}{警告}
Use this iterator on the concurrent container for debugging purpose only. 
\end{DoxyWarning}
