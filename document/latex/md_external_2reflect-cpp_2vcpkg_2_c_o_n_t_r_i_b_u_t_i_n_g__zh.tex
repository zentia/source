Vcpkg 是一个尝试由社区驱动，旨在构建一个多产的、健壮的本地库生态系统 -\/ 您的贡献价值不可估量！\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md752}{}\doxysection{\texorpdfstring{报告问题}{报告问题}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md752}
最简单的方法是通过 {\ttfamily vcpkg.\+exe} 或 \href{https://github.com/Microsoft/vcpkg}{\texttt{ Git\+Hub}} 报告已有的包的问题。 当报告 {\ttfamily vcpkg.\+exe} 的问题时，确保清楚地说明\+:
\begin{DoxyItemize}
\item 机器设置\+: “我用的是\+Windows 10周年更新。 我的机器正位于fr-\/fr区域。 我成功地运行了\textquotesingle{}install boost\textquotesingle{}。”
\item 复现步骤\+: “运行 \textquotesingle{}vcpkg list\textquotesingle{}”
\item 预期结果\+: “我预期看到 \textquotesingle{}boost\+:x86-\/windows\textquotesingle{}”
\item 实际结果\+: “没有输出” 或 “我得到一个崩溃对话框”
\end{DoxyItemize}

当报告包的问题时，一定要清楚地说明\+:
\begin{DoxyItemize}
\item 机器设置 (上述)
\item 您正在构建什么包以及它的版本，例如\+: “opencv 3.\+1.\+0”
\item 构建过程中的任何相关错误日志
\end{DoxyItemize}\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md753}{}\doxysection{\texorpdfstring{贡献 (PR)}{贡献 (PR)}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md753}
我们很乐意接受关于修复、特性、新包和更新现有包的拉取请求。 为了避免浪费您的时间，我们强烈建议您提交一个问题来讨论您想要制作的\+PR是否能被接受。 对于特性和新包来说也是如此。\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md754}{}\doxysubsection{\texorpdfstring{新包贡献准则}{新包贡献准则}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md754}
我们很高兴您有兴趣来提交一个新的包! 这里有一些指导方针来帮助您编写一个优秀的端口文件\+:
\begin{DoxyItemize}
\item 避免功能补丁。 当没有其他方法时，补丁应该被视为实现兼容性的最后手段。
\item 当无法避免补丁时，请不要修改默认行为。 一个补丁的理想生命周期是与上游合并，不再被需要。 在决定如何修补某些内容时，请记住这一目标。
\item 相比原始的 {\ttfamily execute\+\_\+command} 调用，尽量改为通过 {\ttfamily vcpkg\+\_\+xyz} 函数实现。这使得在添加新特性(如自定义编译器标志或生成器)时更容易进行长期维护。
\end{DoxyItemize}\hypertarget{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md755}{}\doxysection{\texorpdfstring{法律声明}{法律声明}}\label{md_external_2reflect-cpp_2vcpkg_2_c_o_n_t_r_i_b_u_t_i_n_g__zh_autotoc_md755}
在您的拉取请求被接受之前，您需要完成一个贡献者许可协议 (CLA)。 本协议证明您允许我们使用您提交的源代码，并且本作品是在合适的许可下提交的，我们可以使用它。

您可以通过 \href{https://cla.microsoft.com}{\texttt{ https\+://cla.\+microsoft.\+com}} 上的步骤来完成\+CLA。 一旦我们收到已签署的\+CLA，我们将审查请求。 您只需要这样做一次。
