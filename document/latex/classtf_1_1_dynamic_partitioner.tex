The partitioner splits iterations into many partitions each of size equal to the given chunk size. Different partitions are distributed dynamically to workers without any specific order.

In addition to partition size, the application can specify a closure wrapper for a dynamic partitioner. \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} closure wrapper allows the application to wrapper a partitioned task (i.\+e., closure) with a custom function object that performs additional tasks.
