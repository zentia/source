\doxysection{Release 3.5.0 (2023/01/05)}
\hypertarget{release-3-5-0}{}\label{release-3-5-0}\index{Release 3.5.0 (2023/01/05)@{Release 3.5.0 (2023/01/05)}}
Taskflow 3.\+5.\+0 is the 6th release in the 3.\+x line! This release includes several new changes, such as pipeline parallelism, improved work-\/stealing performance, profiling, documentation, examples, and unit tests.\hypertarget{release-3-5-0_release-3-5-0_download}{}\doxysubsection{\texorpdfstring{Download}{Download}}\label{release-3-5-0_release-3-5-0_download}
Taskflow 3.\+5.\+0 can be downloaded from \href{https://github.com/taskflow/taskflow/releases/tag/v3.5.0}{\texttt{ here}}.\hypertarget{release-3-5-0_release-3-5-0_system_requirements}{}\doxysubsection{\texorpdfstring{System Requirements}{System Requirements}}\label{release-3-5-0_release-3-5-0_system_requirements}
To use Taskflow v3.\+5.\+0, you need a compiler that supports C++17\+:

\begin{DoxyItemize}
\item GNU C++ Compiler at least v8.\+4 with -\/std=c++17 \item Clang C++ Compiler at least v6.\+0 with -\/std=c++17 \item Microsoft Visual Studio at least v19.\+27 with /std\+:c++17 \item Apple\+Clang Xcode Version at least v12.\+0 with -\/std=c++17 \item Nvidia CUDA Toolkit and Compiler (nvcc) at least v11.\+1 with -\/std=c++17 \item Intel C++ Compiler at least v19.\+0.\+1 with -\/std=c++17 \item Intel DPC++ Clang Compiler at least v13.\+0.\+0 with -\/std=c++17 and SYCL20\end{DoxyItemize}
Taskflow works on Linux, Windows, and Mac OS X.\hypertarget{release-3-5-0_release-3-5-0_summary}{}\doxysubsection{\texorpdfstring{Release Summary}{Release Summary}}\label{release-3-5-0_release-3-5-0_summary}
This release introduces a new data-\/parallel pipeline programming model, solves the busy-\/waiting problem in our work-\/stealing scheduler, and adds a new text-\/based feature for profiler report.\hypertarget{release-3-5-0_release-3-5-0_new_features}{}\doxysubsection{\texorpdfstring{New Features}{New Features}}\label{release-3-5-0_release-3-5-0_new_features}
\hypertarget{release-3-5-0_release-3-5-0_taskflow_core}{}\doxysubsubsection{\texorpdfstring{Taskflow Core}{Taskflow Core}}\label{release-3-5-0_release-3-5-0_taskflow_core}

\begin{DoxyItemize}
\item Added \doxylink{classtf_1_1_worker_interface}{tf\+::\+Worker\+Interface} to allow changing properties of workers upon their creations
\item Added tf\+::\+Executor\+::loop\+\_\+until to allow looping a worker with a custom stop predicate
\item Added \doxylink{classtf_1_1_data_pipeline}{tf\+::\+Data\+Pipeline} to implement data-\/parallel algorithms
\begin{DoxyItemize}
\item See \doxylink{DataParallelPipeline}{Data-\/parallel Pipeline}
\end{DoxyItemize}
\item Extended \doxylink{classtf_1_1_executor}{tf\+::\+Executor} to include \doxylink{classtf_1_1_worker_interface}{tf\+::\+Worker\+Interface}
\item Improved parallel algorithms (e.\+g., \doxylink{classtf_1_1_flow_builder_aae3edfa278baa75b08414e083c14c836}{tf\+::\+Taskflow\+::for\+\_\+each}) with tail optimization
\item Resolved the busy-\/waiting problem in our work-\/stealing algorithm (\href{https://github.com/taskflow/taskflow/pull/440}{\texttt{ \#400}})
\end{DoxyItemize}\hypertarget{release-3-5-0_release-3-5-0_cudaflow}{}\doxysubsubsection{\texorpdfstring{cuda\+Flow}{cuda\+Flow}}\label{release-3-5-0_release-3-5-0_cudaflow}
