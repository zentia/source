In many cases, when you have strings, you know in advance that these strings can only assume a limited number of values. For reasons of efficiency and safety, it is a good idea to use Literals instead of strings.

You can declare a Literal as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{using\ }MyLiteral\ =\ \mbox{\hyperlink{classrfl_1_1_literal}{rfl::Literal}}<\textcolor{stringliteral}{"{}option1"{}},\ \textcolor{stringliteral}{"{}option2"{}},\ ...>;}

\end{DoxyCode}


You can construct literals as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{const}\ \textcolor{keyword}{auto}\ my\_literal\ =\ MyLiteral::make<\textcolor{stringliteral}{"{}option1"{}}>();}

\end{DoxyCode}


Literals that contain only one option also have a default constructor.

\DoxyCodeLine{\ \ \ \ ...}
\DoxyCodeLine{}

\DoxyCodeLine{\ \ \ \ ...}
\DoxyCodeLine{}

\DoxyCodeLine{\}}

\end{DoxyCode}


Literals are stored as {\ttfamily \doxylink{eabase_8h_aba7bc1797add20fe3efdf37ced1182c5}{uint8\+\_\+t}} or {\ttfamily \doxylink{eabase_8h_a273cf69d639a59973b6019625df33e30}{uint16\+\_\+t}} under-\/the-\/hood. But you can extract their name as a string using {\ttfamily .\doxylink{imgui__impl__opengl3__loader_8h_aab18cfce5ee7c6881ae04f18be70d94a}{name()}} or the underlying value using {\ttfamily .\doxylink{imgui__impl__opengl3__loader_8h_abe8e049f756b5ba547bda825af81b645}{value()}}.
