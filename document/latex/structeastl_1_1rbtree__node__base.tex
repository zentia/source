\doxysubsection{详细描述}
\doxylink{structeastl_1_1rbtree__node__base}{rbtree\+\_\+node\+\_\+base}

We define a \doxylink{structeastl_1_1rbtree__node__base}{rbtree\+\_\+node\+\_\+base} separately from \doxylink{structeastl_1_1rbtree__node}{rbtree\+\_\+node} (below), because it allows us to have non-\/templated operations, and it makes it so that the rbtree anchor node doesn\textquotesingle{}t carry a T with it, which would waste space and possibly lead to surprising the user due to extra Ts existing that the user didn\textquotesingle{}t explicitly create. The downside to all of this is that it makes debug viewing of an rbtree harder, given that the node pointers are of type \doxylink{structeastl_1_1rbtree__node__base}{rbtree\+\_\+node\+\_\+base} and not \doxylink{structeastl_1_1rbtree__node}{rbtree\+\_\+node}.
