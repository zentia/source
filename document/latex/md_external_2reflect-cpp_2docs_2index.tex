{\bfseries{reflect-\/cpp}} is a C++-\/20 library for {\bfseries{fast serialization, deserialization and validation}} using reflection, similar to \href{https://github.com/pydantic/pydantic}{\texttt{ pydantic}} in Python, \href{https://github.com/serde-rs}{\texttt{ serde}} in Rust, \href{https://github.com/golang/go/tree/master/src/encoding}{\texttt{ encoding}} in Go or \href{https://github.com/haskell/aeson/tree/master}{\texttt{ aeson}} in Haskell.

reflect-\/cpp fills an important gap in C++ development. It minimizes boilerplate code and enhances code safety for seamless and efficient data exchange across system components.

\doxysubsubsection*{Design principles for reflect-\/cpp include\+:}


\begin{DoxyItemize}
\item Close integration with \href{https://github.com/getml/reflect-cpp?tab=readme-ov-file\#support-for-containers}{\texttt{ containers}} from the C++ standard library
\item Close adherence to C++ idioms
\item Out-\/of-\/the-\/box support for \href{https://rfl.getml.com/supported_formats/json}{\texttt{ JSON}}
\item Simple \href{https://rfl.getml.com/install}{\texttt{ installation}}
\item Simple extendability to \href{https://rfl.getml.com/supported_formats/supporting_your_own_format}{\texttt{ other serialization formats}}
\item Simple extendability to \href{https://rfl.getml.com/concepts/custom_classes}{\texttt{ custom classes}}
\item Being one of the fastest serialization libraries in existence, as demonstrated by our \href{https://rfl.getml.com/benchmarks}{\texttt{ benchmarks}}
\end{DoxyItemize}\hypertarget{md_external_2reflect-cpp_2docs_2index_autotoc_md586}{}\doxysubsection{\texorpdfstring{Why do we need this?}{Why do we need this?}}\label{md_external_2reflect-cpp_2docs_2index_autotoc_md586}
Suppose your C++ program has complex data structures it needs to save and load. Or maybe it needs to interact with some kind of external API. If you do this the traditional way, you will have a lot of boilerplate code. This is annoying and error-\/prone.

!!! example \begin{DoxyVerb}If you are new to reflect-cpp, please refer to our [GitHub repository](https://github.com/getml/reflect-cpp) for the latest list of supported [serialization formats]. You will also find a [simple] and [more comprehensive code example].
\end{DoxyVerb}


reflect-\/cpp is not just a reflection library, it is for {\bfseries{serialization, deserialization and validation}} through reflection.

That means that you can encode your requirements about the input data in the type system and have them validated upfront. This is why the library also includes algebraic data types like tagged unions and numerous validation routines. Having your requirements encoded in the type system is the most reliable way of ensuring they are met. If your requirements are not met, the user of your software gets a very clear error message. Encoding your requirements in the type system also makes it a lot easier for anyone reading your code.

This increases user experience and developer experience, it makes your code safer (fewer bugs) and more secure (less prone to malicious attacks).
