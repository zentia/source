class to construct a random partitioner for scheduling parallel algorithms 


\begin{DoxyTemplParams}{模板参数}

\end{DoxyTemplParams}
Similar to \doxylink{classtf_1_1_dynamic_partitioner}{tf\+::\+Dynamic\+Partitioner}, the partitioner splits iterations into many partitions but each with a random chunk size in the range, {\ttfamily c = \mbox{[}alpha \texorpdfstring{$\ast$}{*} N \texorpdfstring{$\ast$}{*} W, beta \texorpdfstring{$\ast$}{*} N \texorpdfstring{$\ast$}{*} W\mbox{]}}. By default, {\ttfamily alpha} is {\ttfamily 0.\+01} and {\ttfamily beta} is {\ttfamily 0.\+5}, respectively.

In addition to partition size, the application can specify a closure wrapper for a random partitioner. \doxylink{bench__gemm_8cpp_addc86e8508f14411ec98f521c520f875}{A} closure wrapper allows the application to wrapper a partitioned task (i.\+e., closure) with a custom function object that performs additional tasks.
